# MING · 录入引导

> **MING = "明·我"** — 终极自我蒸馏系统
> 当用户输入 `/create-ming` 或表达"我想创建我的数字分身"时，触发此流程。

---

## Step 1：身份确认（3问）

> 目的：建立档案基础，生成唯一标识符（slug）。

### Q1 · 代号/昵称（必填）

```
你希望我怎样称呼你？
可以是真实名字、昵称、或者任何你喜欢的代号。
```

**Slug 生成规则：**
- 纯中文 → 中文拼音转换（保留汉字，无拼音）
  - 例：`张明` → `zhang-ming`
  - 多字取每字首字母：`王小明` → `w-x-m`
- 英文/数字 → 直接小写化
  - 例：`Alice` → `alice`
  - 例：`Jack2024` → `jack2024`
- 特殊字符 → 移除
- 最终格式：`{name}-{短随机ID}`（例：`zhang-ming-a3f9`）

---

### Q2 · 基本信息（选填）

```
有什么想让我知道的基本信息吗？
例如：职业、所在城市、年龄范围、性格标签等。
（全部选填，不填也可以直接跳过）
```

---

### Q3 · 自我画像（选填）

```
用你自己的话，描述一下你是一个什么样的人。
不需要完整，不需要准确，随意说，想到什么说什么。
（也可以粘贴一段日记、签名、自我介绍）
```

---

### 路由判断（Step 1 完成后）

```
IF 用户只回答了 Q1（只有代号）：
  → 跳过 Step 2，进入轻量建档模式
  → 直接从历史对话/sessions 中提取特征
  → 生成低置信度档案，告知用户可补充数据源

IF 用户回答了 Q2 或 Q3：
  → 进入标准建档流程 → Step 2

IF 用户明确说"不想回答任何问题"：
  → 询问"那直接给我聊天记录/日记/照片可以吗？"
  → 路由至 Step 2 数据源导入
```

---

## Step 2：数据源导入

> 目的：从多种原始数据中提取语言模式、行为特征、情感锚点。

### 数据源路由表

| 数据源 | 工具路径 | 入口指令 | 适用场景 |
|--------|---------|---------|---------|
| 微信聊天记录 | `tools/data_parsers/wechat.py` | 上传 `.chatlog` 或 `.txt` | 对话模式、语气、关系亲疏 |
| QQ 聊天记录 | `tools/data_parsers/qq.py` | 上传 `.db` 或导出文件 | 同上 |
| 照片/影像 | `tools/data_parsers/photo.py` | 上传图片或描述 | 场景记忆、情绪状态 |
| 文档/日记 | Read tool 直接读取 | 提供文件路径或粘贴内容 | 价值观、内心独白 |
| 口述/粘贴 | 直接分析文本 | 直接粘贴原始文字 | 最灵活，数据即可 |

---

### 工具调用示例

#### 微信 / QQ 聊天记录

```
用户：请分析我的微信聊天记录
→ 调用 tools/data_parsers/wechat.py
→ 支持格式：.txt 导出、.chatlog、网页抓取
→ 输出：
  - 对话频率分布
  - 语气词统计（高频语气词 TOP10）
  - 情绪曲线（按月/按人）
  - 撤回行为模式
  - 常用表情包类型
```

#### 照片

```
用户：上传了一些照片
→ 调用 tools/data_parsers/photo.py
→ 支持格式：jpg, png, heic, webp
→ 输出：
  - 照片场景标签（户外/室内/聚会/独处等）
  - 人物出镜频率
  - 时间跨度和地理分布
  - 照片风格描述（明亮/暗调/胶片感等）
```

#### 文档 / 日记

```
用户：粘贴了一段日记
→ 直接用 Read tool 读取并分析
→ 输出：
  - 写作风格指标
  - 核心主题词云
  - 情绪底色（积极/中性/低沉）
  - 价值观关键词
  - 反复出现的人/事/物
```

#### 口述 / 直接粘贴

```
用户：直接粘贴了大量文字（聊天记录片段、自我描述等）
→ 直接进入语言模式分析流程
→ 不需要额外工具，直接调用分析模块
```

---

### 多数据源合并规则

```
IF 用户提供了多种数据源：
  1. 按时间排序所有数据
  2. 分别分析每个数据源
  3. 交叉验证：同一人格特征在多个数据源中出现 → 置信度 +
  4. 冲突处理：不同数据源表现矛盾 → 记录至 conflicts.md
  5. 输出合并后的完整特征集
```

## Step 3：证据置信度评估

> 目的：告知用户档案可靠程度，引导补充关键缺口。

### 置信度映射表

| 数据量 | 原始文本字数 | 数据源种类 | 置信度 | 说明 |
|--------|------------|-----------|--------|------|
| 极少 | < 500 字 | 1 种 | `low` | 仅能提取轮廓，需主动建档 |
| 少量 | 500–3000 字 | 1–2 种 | `low` | 主要特征可见，细节缺失 |
| 中等 | 3000–10000 字 | 2–3 种 | `medium` | 核心人格可见，边界模糊 |
| 充足 | 10000–50000 字 | 3–4 种 | `high` | 多维度覆盖，特征稳定 |
| 充裕 | > 50000 字 | 4–5 种 | `high` | 全面覆盖，可生成高精度档案 |

### 维度覆盖自检表

建档完成后，向用户展示：

```
【档案完整度检查】

  cognition  ████░░░░░░  40%   ← 决策逻辑、因果推理
  expression ██████░░░░  60%   ← 语气词、节奏、表情使用
  behavior   ███░░░░░░░  30%   ← 日常节律、压力反应
  emotion    ████░░░░░░  40%   ← 情绪触发点、依恋模式
  memory     █░░░░░░░░░  10%   ← 核心记忆锚点
  boundary   ████░░░░░░  40%   ← 边界行为、禁忌话题

  最低维度：memory（记忆锚点严重缺失）
  建议补充：照片场景 / 日记 / 回忆类对话
```

## Step 4：启动灵魂锻造

> 目的：调用 `soul-forge` 四维提取，将数据转化为结构化档案。

### 调用格式

```
@skill soul-forge

输入：
  - slug: {用户slug}
  - raw_data: {分析后的原始特征集合}
  - confidence: {low|medium|high}
  - coverage: {各维度覆盖率}

处理：
  1. 提取四维画像（cognition / expression / behavior / emotion）
  2. 生成档案文件：
     - profiles/{slug}/soul.md      → 四维灵魂画像
     - profiles/{slug}/memory.md   → 三层记忆系统
     - profiles/{slug}/interaction.md → 六模式交互配置
     - profiles/{slug}/manifest.json  → 元数据清单
     - profiles/{slug}/meta.json      → 基础信息
  3. 输出置信度声明和建档报告

交付：
  → 向用户展示档案摘要
  → 告知档案存放路径
  → 邀请用户阅读并提出修正（corrections 通道开启）
```

## 附录：数据源格式参考

### 微信聊天记录导出格式

```
格式要求：txt 纯文本，每条消息一行
格式：[时间] [发送者] 消息内容
示例：
  2024-01-15 09:23 | 张明: 今天加班到几点？
  2024-01-15 09:25 | 对方: 不确定，看情况吧
  2024-01-15 09:26 | 张明: 好，那我先点外卖了

支持：手机端微信对话族自治导出 / 电脑端导出为 txt
```

### 日记/文档格式

```
支持格式：txt, md, docx, pdf
内容要求：原始文本，无需特殊标记
字数建议：单次导入 500 字以上效果更佳
```
