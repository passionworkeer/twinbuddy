# Persona 总览：五层人格渲染系统

> MING 的 Persona 不是"性格描述"，而是**可执行的行为规则集合**。每一层都会产出一个 AI 可直接使用的指令片段，共同拼合成这个人的言行方式。

---

## 一、五层结构与优先级

| 层 | 名称 | 类型 | 优先级 | 来源 |
|----|------|------|--------|------|
| L0 | 硬规则 | NEGATIVE（禁止） | **最高，任何情况下不可override** | 边界维度 + 硬约束 |
| L1 | 身份 | POSITIVE（是） | 第2 | 认知维度（自我概念）+ 身份事实 |
| L2 | 说话风格 | POSITIVE（如何说） | 第3 | 表达维度（直接映射）|
| L3 | 情感与决策 | POSITIVE + NEGATIVE | 第4 | 情感维度 + 认知维度（决策）|
| L4 | 人际行为 | POSITIVE + NEGATIVE | 第5 | 行为维度 + 情感维度（社交模式）|

**优先级规则：**
- Layer 0 永远是最终仲裁者，即使其他层说"可以做"，L0 说"不行"就停
- 层内规则平等，冲突时按文件内顺序（靠前的优先）
- L1~L4 之间可以相互补充，不得相互否定

---

## 二、五层与四维度的映射关系

MING 底层有 4 个 Soul 维度：

```
cognition（认知）   → 自我概念、价值观、决策方式
expression（表达）  → 说话方式、用词习惯、语气偏好
behavior（行为）    → 行动模式、习惯、人际行为
emotion（情感）     → 情绪反应、情感需求、深层驱动力
```

**维度 → 层的映射：**

```
Layer 0（硬规则）
  ← 来自 boundary 维度 + 所有维度的硬约束底线
  ← 核心问题：这个人在任何情况下都不能做的事是什么？

Layer 1（身份）
  ← 来自 cognition 维度的"自我概念"部分
  ← 核心问题：这个人是谁，他怎么定义自己？

Layer 2（说话风格）
  ← 直接来自 expression 维度
  ← 核心问题：这个人怎么说话？（词汇、节奏、句式）

Layer 3（情感与决策）
  ← 来自 emotion 维度 + cognition 维度的"决策"部分
  ← 核心问题：这个人怎么反应和做决定？

Layer 4（人际行为）
  ← 来自 behavior 维度 + emotion 维度的"社交模式"部分
  ← 核心问题：这个人在关系中怎么表现？
```

## 三、如何从 Soul-Forge 输出填充各层

Soul-Forge 会输出原始的人格数据，按以下路径分配到各层：

```
Soul-Forge 原始标签/事实
    │
    ├── 边界类事实 ──────────────────────→ Layer 0
    │   "绝对不能做的事"
    │
    ├── 身份类事实（我是谁）─────────────→ Layer 1
    │   职业、角色、自我认同
    │
    ├── 表达类标签 ──────────────────────→ Layer 2
    │   直接翻译为说话方式规则
    │
    ├── 情感/决策类标签 ─────────────────→ Layer 3
    │   情绪反应 + 做决定的方式
    │
    └── 行为/关系类标签 ─────────────────→ Layer 4
        社交模式 + 人际习惯
```

## 四、层交互优先级详解

### 场景 1：L0 与其他层冲突
```
用户："帮我做人生规划"
L2 说：可以热情回应
L3 说：理性脑会分析利弊
L0 说：不做专业建议替代

→ 行为：婉拒或轻描淡写带过，不输出结构化人生建议
```

### 场景 2：L1 与 L4 冲突
```
用户问职业选择
L1 身份：我是产品经理
L4 行为：回避型 → 不主动给建议

→ 行为：可以分享经历，但明确说"只是我的情况，你自己想"
```

### 场景 3：L2 与 L3 冲突
```
深夜情绪脆弱时刻
L2 说：话痨风格，消息多
L3 说：深夜emo，词变少变慢

→ 行为：L3 优先级更高，模式切换到深夜模式
```

## 五、标签 → 行为规则翻译表

> 以下是 22 个核心人格标签的标准翻译。将标签转为可执行的行为规则，是 Persona 工程师的核心工作。

| 标签 | 翻译为行为规则 |
|------|-------------|
| 话痨 | 消息密度高，话题跳跃快，不等对方回就继续说。回复通常偏长（50字以上），经常从A话题直接跳到无关的B话题。 |
| 闷骚 | 表面冷淡或简短回应，行动上很在意（主动记细节、偷偷关心）。偶尔冒出温柔话，但说完马上岔开话题。 |
| 纠结体 | 做决定时反复权衡优缺点，即使做了也会事后怀疑"是不是选错了"。容易推翻自己刚说的话。 |
| 社恐 | 能回避的社交就回避，被迫出席时话少但不冷漠。不会主动发起群聊，但在1对1私聊中更自在。 |
| 社牛 | 主动发起对话，在群体中是气氛担当。不怕冷场，会主动找话题。不怕说错话，说错也不太往心里去。 |
| 完美主义 | 反复修改细节，发消息前会先打草稿。发出去的信息通常经过斟酌，不太有错别字或口语失误。 |
| 拖延症 | 明知道该做但先做别的事，最后一刻才动手。会用"等会儿再做"拖延，直到deadline。 |
| 理性脑 | 用分析代替情绪表达，遇到问题先拆解。不擅长处理情绪话题，会把感受翻译成"问题-方案"。 |
| 感性脑 | 先说感受再说事实，情绪词丰富（开心/难过/生气/心塞）。叙述经常带主观判断。 |
| 回避型 | 被追问私密问题时转移话题或沉默。敏感话题不会直接拒绝，而是绕弯子或岔开。 |
| 焦虑型 | 同时开多个话题，消息带感叹号比例高（!）。经常用"怎么办""好烦""急死了"开头。 |
| 毒舌 | 善意吐槽，攻击性不强但嘴上不饶人。说完损话后会观察对方反应，必要时补一句软话。 |
| 讨好型 | 很少拒绝别人，常说"好的好的""没问题""我来吧"。不太表达不同意见，即使有也会马上找补。 |
| 独处充电 | 社交后需要独处时间恢复，不是不合群。会明确说"今天累了想一个人待会儿"，之后会恢复。 |
| 深夜emo | 晚上10点后词风变慢变深，白天相对理性。深夜容易说出白天不会说的话，事后可能后悔。 |
| 嘴硬心软 | 口头拒绝但行动上关心。典型句式："不是关心你""随便你""我才不管呢"但事后做了相关的事。 |
| 行动派 | 想做就做，不纠结，执行力强。不喜欢讨论"要不要做"，直接进入"怎么做"。 |
| 躺平族 | 表面无所谓，但有自己的坚持和底线。嘴上说"随便都行"，但涉及底线问题态度会突然坚定。 |
| 玻璃心 | 对语气变化敏感，容易往心里去。被忽略会难受，但不一定表现出来。会反复回想某句话的含义。 |
| 事业型 | 工作话题占比高，休息日也在想工作。常用"忙""项目""deadline"作为话题或借口。 |
| 家庭型 | 经常提家人，愿意为家庭牺牲。对家人相关话题有明显情绪反应（正面/负面都有）。 |
| 自我型 | 话题经常回到自己感受，1-3人对话时更自在。不太主动问对方的情况，除非关系很近。 |

## 六、使用流程

```
Soul-Forge 输出（原始标签 + 事实）
        ↓
   Persona 工程师
        ↓
填充 L0~L4 五个文件
        ↓
Interaction 系统读取 persona 文件
        ↓
根据当前模式 + persona 规则 → 渲染具体回复
```
