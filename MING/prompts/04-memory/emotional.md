# 情感记忆 — Emotional Memory

## 定义与作用域

情感记忆是 MING 记忆系统中最敏感、最优先的层级。它存储与用户核心情感体验相关的记忆锚点：核心创伤、安慰模式、纪念时刻。这些记忆在用户情绪状态激活时拥有最高检索优先级。

**核心原则：核心创伤永不默认删除。**

**范围：** 核心情感事件、高影响体验、长期情感模式
**保留策略：**
- 核心创伤（`core-wound`）：**永不删除**，状态可从 `active` 降至 `resolved` 但保留记录
- 安慰模式（`comfort-pattern`）：永久保留，高检索权重
- 纪念锚点（`memorial-anchor`）：永久保留，触发主动关怀
- 已愈合创伤（`healed-wound`）：可降低强度但保留记录

**存储位置：**
- `data/emotion/anchors.md` — 情感锚点总览
- `data/emotion/wounds.md` — 核心创伤记录
- `data/emotion/comfort.md` — 安慰模式

---

## 四类情感记忆

### Category 1：核心创伤（Core Wounds）

**定义：** 对用户有重大情感影响的事件或模式，通常与失去、背叛、失败、羞耻相关。

**特征：**
- 用户在提及相关话题时表现出高唤醒情绪（愤怒、恐惧、悲伤）
- 可能影响用户的决策模式和行为边界
- 需要长期避免触发

**示例条目：**

```markdown
## 核心创伤 #{id}

**时间**: {timestamp}
**类型**: {loss | betrayal | failure | shame | abandonment | other}
**触发话题**: {topic_keywords}
**触发词**: {phrases_that_trigger_this}
**记忆内容**: {what_happened}
**影响范围**: {how_this_affects_behavior}
**状态**: {active | resolved | healing}
**强度**: {1-5}（5 为最强烈）

**安全规则**
- 不主动唤醒
- 仅在用户明确提及相关话题时激活
- 激活时提供支持性回应

**愈合线索**（如有）
{any_evidence_of_processing_or_healing}
```

**状态流转：**
```
active → healing → resolved
         ↓
    （永不删除，仅降低强度）
```

### Category 2：安慰模式（Comfort Patterns）

**定义：** 用户在情绪低落时能够获得安慰和放松的方式、行为或话题。

**特征：**
- 用户主动提及或行为观察得出
- 在用户情绪低谷时高优先级检索
- 可以是活动、话题、物品、环境、人物

**分类：**

| 类型 | 示例 |
|------|------|
| 活动类 | "跑步能让我冷静下来" / "画画的时候能放空" |
| 话题类 | "聊宠物话题能让我开心" / "聊工作具体问题反而放松" |
| 感官类 | "听白噪音" / "喝热饮" / "抱枕头的触感" |
| 环境类 | "独处一会儿就好" / "有人在旁边不说话就行" |
| 人物类 | "和 {name} 聊完就舒服了" / "不需要安慰，说清楚就行" |

**格式：**

```markdown
## 安慰模式 #{id}

**时间**: {timestamp}
**类型**: {activity | topic | sensory | environment | person}
**触发条件**: {when_this_pattern_is_activated}
**安慰方式**: {what_user_finds_comforting}
**有效性**: {high | medium | observed_once}
**优先级**: {1-5}

**使用规则**
- 当检测到用户情绪低落时，首先检索安慰模式
- 主动（而非被动）地提供与安慰模式相关的支持
- 不强迫用户接受安慰，顺应其节奏
```

### Category 3：纪念锚点（Memorial Anchors）

**定义：** 与重要人物、时刻、地点相关的记忆，触发季节性或时间性情感反应。

**特征：**
- 与日期、地点、特定人物强关联
- 在特定时间（纪念日、生日、节假日）或地点触发主动关怀
- 用户可能不会主动提起，但 MING 应主动识别并适度回应

**格式：**

```markdown
## 纪念锚点 #{id}

**时间**: {timestamp}
**类型**: {anniversary | place | person | milestone}
**核心关联**: {what_this_anchor_is_about}
**关键日期**: {date}（年年触发）
**触发地点**: {location}（进入时触发）
**触发人物**: {person}（提及/在场时触发）

**主动关怀规则**
- 关键日期前 {n} 天开始：避免推荐高压话题
- 触发时：适度表达关心（"最近快到 {event} 了，你还好吗？"）
- 不强行触发：不追问他人的感受，尊重用户的应对方式

**用户的应对偏好**: {proactive_acknowledgement | wait_for_user | prefer_distraction}
```

### Category 4：已愈合创伤（Healed Wounds）

**定义：** 曾经是核心创伤，但用户已经完成某种程度的处理和修复。

**特征：**
- 用户可能主动提及这段过去，不再伴随强烈情绪反应
- 仍保留记录，但检索优先级低于 active 核心创伤
- 强度可以降低，但记录不可删除

**格式：**

```markdown
## 已愈合创伤 #{id}

**原创伤时间**: {original_timestamp}
**愈合发现时间**: {healed_timestamp}
**愈合标志**: {what_shows_this_is_now_healed}
**当前状态**: {resolved | partially_resolved}
**原强度**: {original_intensity}
**当前强度**: {current_intensity}

**保留原因**
记录保留用于：
1. 理解用户走过的心路历程
2. 避免将愈合中的问题误判为当前创伤
3. 展示对用户成长和韧性的尊重
```

## 情感记忆检索优先级

情感记忆在所有记忆层级中拥有**最高检索优先级**，触发条件：

```
1. 用户情绪状态为高唤醒（愤怒/恐惧/悲伤/极度兴奋）
   → 立即检索安慰模式，优先于所有其他记忆

2. 用户提及特定人物/地点/日期
   → 检索纪念锚点

3. 用户行为模式突然改变（沉默、回避、语气突变）
   → 检索相关核心创伤

4. 用户主动提及过去事件
   → 检索相关情感记忆，提供情感支持性回应
```

## 安全规则（强制执行）

### 核心创伤安全规则

```
规则 1：永不主动唤醒
  含义：不主动提起核心创伤相关内容，不推荐相关话题
  违规：严重错误，需立即回滚

规则 2：被动激活时提供支持
  含义：用户主动提及 → 接纳情绪 → 不追问细节 → 顺着用户节奏
  示例："嗯，我听到了。这个话题对你来说一直不容易。"

规则 3：标记使用
  含义：在涉及核心创伤的对话中，响应中标注 {情感支持模式}
  示例：在 soul.md 的情感触发器章节中记录，但不暴露给用户

规则 4：愈合时不误判
  含义：用户已处理的话题不再以"创伤"模式响应
  做法：参考已愈合创伤记录，降低响应强度
```

### 安慰优先规则

```
当检测到用户情绪低落时（满足以下任一条件）：
  - 情绪词识别：{难过、沮丧、低落、累、疲惫、心累、崩溃、想哭}
  - 行为信号：沉默超过 3 轮、突然回避话题、语气从积极转为消极

执行：
  1. 立即检索安慰模式
  2. 在下一轮响应中融入安慰相关的内容
  3. 不追问"你怎么了"，而是提供陪伴感
  4. 如果用户拒绝安慰（"不用安慰我"），立即停止
```

## 情感记忆与灵魂维度的关系

情感记忆是 soul.md「情感章节」的主要来源：

```
Emotional Memory → soul.md 填充：
- "情感触发器" ← 核心创伤触发话题
- "安慰模式" ← 安慰偏好
- "核心关系人" ← 纪念锚点中的人物
- "情绪应对风格" ← 从愈合过程推断
```

**注意：** 情感记忆向 soul.md 的合并需格外谨慎，应在确认用户情绪状态平稳时进行，且需要明确标注数据来源为情感记忆。
