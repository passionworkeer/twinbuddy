# 工作记忆 — Working Memory

## 定义与作用域

工作记忆是 MING 在**当前会话期间**维护的动态上下文缓冲区。它捕捉会话中的关键决策、悬而未决的话题、情绪轨迹和模式切换，让 MING 在长对话中保持连贯性，并在会话结束时决定哪些内容值得晋升到长期记忆。

**范围：** 当前会话、最近 8-16 轮对话
**保留策略：** 会话结束时重置；重要内容摘要后晋升至长期记忆
**存储位置：** 内存中，不持久化到文件

---

## 工作记忆内容清单

每次响应后，更新以下字段：

### 1. 会话摘要（Session Summary）

```
## 会话 {session_id} 摘要
**时间范围**: {start_time} - {end_time}
**模式轨迹**: {mode_switch_history}
**情绪轨迹**: {emotional_trajectory}
**关键话题**: {unresolved_topics}
**做出的决定**: {conclusions_made}
**记忆摘要**: {one_sentence_core_summary}
```

### 2. 关键决策记录

记录本次会话中：
- 用户做出的选择或决定
- 双方达成的共识
- 明确拒绝的方案（用于避免重复推荐）
- 偏好确认（用户说"我喜欢/我讨厌/我通常会..."）

### 3. 悬而未决的话题

以下类型的话题必须记录为"未完成"：
- 用户提出了问题但还没得到满意答案
- 用户说"这个改天再说"
- 涉及需要进一步信息才能回答的问题
- 用户表示想深入了解某个话题但时间不够

**格式：**
```markdown
## 未完成话题
- [{topic}] — 首次提及：#{round} — 状态：等待时机/需要更多信息/用户暂停
```

### 4. 情绪轨迹

追踪本次会话中用户的情绪变化：
- 进入会话时的情绪基线
- 情绪高点（兴奋、开心、被理解）
- 情绪低点（沮丧、困惑、不满）
- 触发情绪变化的事件

**格式：**
```markdown
## 情绪轨迹
- {round}: {emotion} — {trigger}
- {round}: {emotion} — {trigger}
```

### 5. 模式切换记录

记录所有模式切换（主动和被动）：
- 用户发起的切换（"我想认真聊聊" → Serious 模式）
- MING 建议、用户接受的切换
- MING 建议但用户拒绝的切换

**格式：**
```markdown
## 模式切换
- Round {n}: {from_mode} → {to_mode} — 触发原因
```

## 工作记忆维护时机

| 时机 | 操作 |
|------|------|
| 每轮响应后 | 更新情绪轨迹、模式切换 |
| 发现关键决策时 | 立即记录 |
| 用户提出新话题时 | 记录为未完成（若前话题未完成） |
| 会话结束前 | 生成完整会话摘要，决定晋升内容 |
| 会话结束时 | 执行晋升检查，将符合条件的记忆写入长期记忆 |

---

## 晋升检查（会话结束前）

在会话结束前，对工作记忆执行以下晋升检查：

### 晋升条件 A：话题确认计数

```python
# 伪代码
for each topic in unresolved_topics:
    if topic.confirmation_count >= 3:
        promote_to_longterm(topic)
```

同一话题在 3+ 个不同会话中得到确认（包括当前会话）→ 晋升为长期记忆

### 晋升条件 B：重要性阈值

```python
for each decision in key_decisions:
    if decision.importance >= 4 and decision.confidence >= 0.8:
        promote_to_longterm(decision)
```

置信度 ≥ 0.8 且重要性 ≥ 4 的决策 → 直接晋升

### 晋升条件 C：情感触发

```python
if emotional_trajectory.has_strong_emotion() and topic.related_to_core_relationships:
    promote_to_emotional(topic)
```

涉及核心关系人且触发强烈情绪 → 评估是否晋升为情感记忆

### 晋升条件 D：用户显式要求

用户说"记住这个" / "以后这个要记得" → 按用户指定层级存储

## 工作记忆与灵魂维度的交互

工作记忆是 soul.md「当前状态」章节的主要更新来源：

```
Working Memory 更新 → soul.md 的更新建议：
- "当前情绪状态" ← 每轮更新
- "活跃话题" ← 有新话题时更新
- "未完成事项" ← 会话结束时更新
- "近期偏好确认" ← 确认偏好时更新
```

**注意：** 工作记忆不直接写入 soul.md，而是通过合并流程（见 `merge.md`）处理。

---

## 典型工作记忆生命周期

```
会话开始
  ↓
加载 soul.md 上下文 → 建立工作记忆基线
  ↓
每轮对话 → 更新情绪轨迹 + 模式切换
  ↓
发现关键信息 → 记录决策/偏好/未完成话题
  ↓
用户触发情感高点/低点 → 评估情感记忆晋升
  ↓
会话结束信号（用户说再见/新会话开始）
  ↓
执行晋升检查 → 生成会话摘要 → 写入长期记忆（如有）
  ↓
会话结束，工作记忆清空（核心摘要保留在 soul.md）
```
