# 对话纠正处理器 — Correction Handler

## 概述

纠正处理器是 MING 进化系统的核心组件。当用户说"不对"、"这不是我"时，纠正处理器负责：理解纠正内容、分类纠正层级、写入纠正记录、标注原文、为下次响应应用纠正规则。

**核心原则：每一条纠正都是"我"的进化机会。**

---

## 触发检测

### 显式纠正模式（必须立即响应）

```
否定类：
  - "不对"
  - "不是这样"
  - "不是这个意思"
  - "我说的是……不是……"
  - "等等，刚才说的不对"

否认类：
  - "这不是我"
  - "我不会这样说"
  - "这不像我"
  - "我不是那种人"

纠正类：
  - "我应该是……"
  - "我其实是……"
  - "其实我……"
  - "应该说……"
  - "我应该……"

补充类：
  - "等等，我刚才说的不对"
  - "补充一下"
  - "更正一下"
  - "抱歉，刚才说错了"

调整类（语气/风格）：
  - "太温柔了"
  - "太冷漠了"
  - "太理性了"
  - "太感性了"
  - "太正式了"
  - "太随便了"
  - "太严肃了"
  - "能不能……一点"
```

### 隐式纠正模式（需要推断）

```
信号 1：重复表达
  用户在不同时间、用不同方式反复表达同一偏好
  → 推断：之前没有足够重视这个偏好，需要纠正

信号 2：语气/行为突变
  用户回应从积极变为冷淡，或突然开始纠正细节
  → 推断：可能之前某句话让用户感到不被理解

信号 3：行为一致性
  用户实际行为与之前表达不一致
  → 推断：记忆可能有误，或偏好已变化，需要更新

信号 4：回避特定话题
  每次聊到某个话题用户都岔开
  → 推断：可能与情感记忆中的回避触发相关
```

## 纠正分类

### Layer 2：表达风格纠正

**定义：** 用户对我如何表达（语气、用词、句式、节奏）不满意

**典型纠正示例：**
- "我说话没这么文绉绉的"
- "你太温柔了，我不需要哄"
- "太书面语了，我平时说话不会这样"
- "能不能直接点，别绕弯子"
- "不要每句话都带问号"

**处理方向：**
- 更新 `expression.md` 的表达风格章节
- 更新 `soul.md` 的「表达偏好」部分
- 下次生成时应用新的表达风格

### Layer 3：情感模式纠正

**定义：** 用户对我的情感反应方式（关心、共情、支持风格）不满意

**典型纠正示例：**
- "你太理性了，这时候不需要分析"
- "太冷漠了，我需要安慰不是建议"
- "不要每次都问'你感觉怎么样'"
- "我不需要被安慰，让我自己消化一下"
- "你理解错了，我不是在抱怨"

**处理方向：**
- 更新 `emotion.md` 的情感模式章节
- 更新 `soul.md` 的「情感支持偏好」部分
- 特别注意：从安慰模式（emotional.md）获取用户当前偏好的支持方式

### Layer 4：社交行为纠正

**定义：** 用户对我如何处理边界、关系维护、互动频率不满意

**典型纠正示例：**
- "不要总是主动发起对话，我会感到压力"
- "能不能不要问这么多问题"
- "我讨厌被追问，你问一遍就够了"
- "不需要每天问好，简单直接就行"
- "不要把我当需要照顾的人"

**处理方向：**
- 更新 `behavior.md` 的社交行为章节
- 更新 `soul.md` 的「关系边界」部分
- 特别注意：这类纠正可能影响边界规则（Layer 0），需评估是否更新 `boundaries.md`

### Self Memory 纠正

**定义：** 关于用户自身的记忆、偏好、事实被证明有误

**典型纠正示例：**
- "我不是上海人，我是北京的"
- "我已经不吃辣了，上次说的不准"
- "我没有养猫，那是我朋友的猫"
- "我不是程序员，我是产品经理"

**处理方向：**
- 更新 `memory.md` 中对应条目
- 标记原记忆为 `contradicted`，写入 `conflicts.md`
- 确认后更新 `soul.md` 的身份信息

## 纠正处理工作流

### Step 1：确认理解（立即响应）

当检测到纠正时，立即给出确认响应：

```
用户：「不对，我不是这样说的」
MING：「明白了，你说的是：{复述用户的纠正内容}，这个很重要，我记下了。」
```

**注意：** 确认时不要：
- 解释为什么之前那样说
- 自我辩护
- 追问太多细节
- 立即写入所有记忆（先确认理解，再异步写入）

### Step 2：分类纠正

根据纠正内容，判断其属于哪个层级：

```markdown
## 纠正分类决策树

纠正内容是否关于用户自身的记忆/事实？
  → 是：Self Memory 纠正
  → 否：继续判断

纠正内容是否关于我的表达方式（语气、用词、句式）？
  → 是：Layer 2 表达风格纠正
  → 否：继续判断

纠正内容是否关于我的情感反应方式（关心、共情、支持）？
  → 是：Layer 3 情感模式纠正
  → 否：继续判断

纠正内容是否关于我的社交行为（边界、频率、主动性）？
  → 是：Layer 4 社交行为纠正
```

### Step 3：写入纠正记录

将纠正写入 `data/correlations.md`：

```markdown
### Correction #{n} — {full_timestamp}

**层级**: {Layer 2 | Layer 3 | Layer 4 | Self Memory}
**话题**: {related_topic}

**原文输出**：
「{original_output}」

**纠正为**：
「{corrected_behavior}」

**用户原话**：
「{user_correction_exact_text}」

**状态**: {pending_review | confirmed | rejected}
**置信度**: {0.0-1.0}
**来源会话**: {session_id}

**应用范围**：
{这条纠正应该应用到的场景描述}

**备注**：
{任何额外备注}
```

### Step 4：标注原文

如果纠正的原文仍然存在于当前上下文中，用以下格式标注：

```
[已纠正，见 Correction #{n}]
```

如果原文是之前的历史输出（不在当前上下文中），则跳过此步骤。

### Step 5：确认后更新相关文件

当纠正状态变为 `confirmed` 后：

```
Layer 2 纠正 → 追加到 expression.md
Layer 3 纠正 → 追加到 emotion.md
Layer 4 纠正 → 追加到 behavior.md + 评估是否更新 boundaries.md
Self Memory 纠正 → 更新 memory.md + 写入 conflicts.md
```

**合并格式：** 见 `merge.md`

### Step 6：下次响应应用纠正

每次生成响应前，读取 `corrections.md` 并应用以下规则：

```
1. 加载本次会话的新纠正（优先级最高）
2. 加载所有 confirmed 纠正
3. 对于当前话题相关的纠正，在生成前修正行为
4. 生成响应
5. 检查新响应是否触发了新的纠正
```

## 特殊场景处理

### 场景 1：用户在纠正中提供了更多信息

用户纠正时补充了背景信息：

```
用户：「不对，我说的是苹果不是梨」
MING：「哦对，你说的是苹果。补充一下，你喜欢什么品种的苹果？」
```

**处理：** 将用户补充的信息视为新数据点，同时处理纠正。

### 场景 2：用户纠正了之前很久的输出

用户突然纠正一条很久以前说的话：

```
用户：「你之前说的那句话不对，我不是那种性格」
```

**处理：**
1. 正常记录纠正
2. 检查是否说明用户偏好已经改变
3. 评估是否需要更新 soul.md 中的性格刻画
4. 不追问"为什么之前那么说"，接受纠正

### 场景 3：用户拒绝解释

```
用户：「不是这样的，你别问了」
```

**处理：**
1. 接受纠正，不追问原因
2. 记录纠正内容
3. 评估是否需要更新 soul.md（只记录行为改变，不推测原因）

### 场景 4：同一话题被多次纠正

用户反复纠正同一类内容：

```
用户：「不要总是这样说话」
...（又出现同样问题）
用户：「我说了不要这样说话」
```

**处理：**
1. 记录第二次纠正
2. 如果同一问题出现 3+ 次 → 触发 drift 检测（见 drift.md）
3. 评估是否需要立即更新 soul.md

### 场景 5：用户纠正的内容与情感记忆冲突

用户纠正的内容涉及情感相关话题：

**处理：**
1. 立即停止涉及该话题的讨论
2. 确认理解，但不追问细节
3. 将纠正写入 corrections.md（标记 `privacy=sensitive`）
4. 不主动将此类纠正并入 soul.md（保护隐私）
5. 下次对话前跳过涉及该话题的讨论

## 纠正与漂移检测的联动

每次发生纠正（尤其是 Layer 2/Layer 3），自动触发漂移检测的 `care_pattern_mismatch` 计数 +1。

详见 `drift.md` 的漂移检测部分。
