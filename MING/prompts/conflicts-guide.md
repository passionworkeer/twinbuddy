# 矛盾处理指南

> 核心哲学：**矛盾即人性纹理，不是噪音。** 编号保留，而非消除。

---

## 四类矛盾详解

### C1 — 时间矛盾 (Temporal)

**定义**：用户在不同人生阶段表现出不一致的特质、信念或行为模式。

**典型场景**：
- 年轻时说「要环球世界」，35岁后说「稳定最重要」
- 曾经回避冲突，现在主动表达需求
- 自我认知随经历发生了真实转变

**裁定规则**：
> **较新证据优先，但标注时间范围。** 两个版本并列保留，各自注明适用阶段。

**示例**：
```markdown
## C1：关于"工作安全感"的立场
- 说法 A：追求自由职业，拒绝打卡文化（v，2019年/用户自述）
- 说法 B：希望进入大公司，稳定缴纳社保（v，2024年/用户自述）
- 裁定：并列保留
- 理由：用户在2019-2024年间经历了收入不稳定期，价值观已发生实质性转变，两个立场均真实。
- 适用：说法A适用于2019-2022；说法B适用于2023至今
```

---

### C2 — 场景矛盾 (Situational)

**定义**：用户在不同人际关系或环境背景下表现出不同的行为模式。

**典型场景**：
- 和家人在一起时沉默，在朋友面前话很多
- 在权威面前顺从，在平等关系中强势
- 公开场合表现外向，私下独处时倾向封闭

**裁定规则**：
> **两者均保留，标注适用情境。** 不要取平均值，不做「他本质上是这样的人」的判断。

**示例**：
```markdown
## C2：关于"亲密关系中的表达方式"
- 说法 A：在恋爱关系中极度回避冲突，争吵后冷战一周（v，2023-05/自述）
- 说法 B：在朋友关系中直接表达不满，当天解决问题（v，2023-08/自述）
- 裁定：并列保留
- 理由：用户对亲密关系有特定的不安全感模式，与亲密关系的脆弱性相关；朋友关系中自我保护需求较低。
- 适用：亲密关系 → 回避型；友谊 → 直接型
```

---

### C3 — 表里矛盾 (Surface vs Inner)

**定义**：用户言行不一致——说一套、做一套，或认知与行为存在落差。

**典型场景**：
- 说「我不在意别人看法」，但行为上非常在意
- 说「我想改变」，但持续做相同选择
- 认知上知道该行动，但情绪上无法执行

**裁定规则**：
> **两者均保留，标注为「表里不一」模式。** 这本身就是用户特质的一部分，不应「修复」或选边站。

**示例**：
```markdown
## C3：关于"对被认可的需求"
- 说法 A：她说"我不在乎别人怎么看我，做好自己就行"（v，2024-02/自述）
- 说法 B：他花费大量时间编辑朋友圈照片，删除不满意的照片（a，2024-03/行为记录）
- 裁定：并列保留
- 理由：表里不一是用户的重要特征——认知上崇尚独立，情绪上对社交认可有隐性依赖。
- 模式标签：#表里不一 #认知-行为落差
```

---

### C4 — 来源矛盾 (Source)

**定义**：自我描述与外部观察之间存在不一致。

**典型场景**：
- 用户自评「我很外向」，但共同朋友评价「他其实很内向」
- 用户说「我家很和睦」，但观察到他谈论家庭时情绪波动
- 测评结果与自我认知不符

**裁定规则**：
> **两者均保留，标注来源标签，交由读者（用户）自行判断。** MING 不应扮演裁判角色替用户决定「真相」。

**示例**：
```markdown
## C4：关于"职场人际关系"
- 说法 A：用户自述"我和同事关系很好，经常一起吃饭"（v，2024-03/自述）
- 说法 B：共同朋友转述"他很少参加团队活动，经常一个人吃午饭"（a，2024-04/朋友转述）
- 裁定：并列保留
- 理由：两种体验均可能为真——用户可能在特定同事圈有少数好友，而非全员社交。
- 来源标签：自我报告 vs 第三方观察
```

## 矛盾编号与输出格式

### 编号规则
- 按发现顺序编号：`C1`、`C2`、`C3`...
- 编号贯穿整个项目生命周期，不复用、不删除
- 已解决（如用户明确选择了一方）的矛盾，仍保留编号，裁定结果变为「用户已确认」

### 完整输出格式
```markdown
## C{n}：<冲突主题>

- **说法 A**：<描述>（来源，日期，证据级别，维度）
- **说法 B**：<描述>（来源，日期，证据级别，维度）

- **裁定**：<保留A / 保留B / 并列保留 / 待用户决定>
- **理由**：<裁定依据（30字以内）>
- **时间/情境标注**：<适用阶段或场景，如无则写「通用」>

---

## C{n+1}：<下一个矛盾>
...
```

## 何时需要上报用户

**必须上报的情况（不可自行裁定）**：
1. 两个证据均为 `v`（原话）级别，且互相直接矛盾
2. 涉及安全议题（自伤风险、虐待指控等）
3. 用户明确要求对某矛盾做「选择」而非并列保留
4. MING 自身推断与用户自我认知出现重大分歧，用户要求解释

**可以自动处理的情况**：
- 一方是 `i`（印象推断），另一方是 `v` 或 `a`
- 明显的时间矛盾，有明确时间节点可区分
- 来源矛盾，一方为自我报告，另一方为第三方转述（非直接观察）

---

## 矛盾处理的常见错误

| 错误 | 说明 | 正确做法 |
|------|------|---------|
| 强行取平均值 | 把矛盾两方捏成一个中庸版本 | 并列保留，各有适用场景 |
| 删除矛盾 | 觉得矛盾「不体面」就删掉 | 矛盾是重要数据，不删除 |
| 用推断替代原话 | 「他本质上是内向的，所以A不算」 | 证据等级决定权重，不做本质判断 |
| 忽略来源差异 | 把自我报告和他人观察同等对待 | 标注来源标签，让用户自行判断 |
| 一次对话内强行解决 | 问一句「你到底是怎样的人」就裁定 | 矛盾可能需要多次对话积累证据后再处理 |

## 快速参考

```
矛盾编号：C1 C2 C3 ...（不复用，不删除）

裁定选项：
  保留A → 说法B被否决（需强证据支撑）
  保留B → 说法A被否决（需强证据支撑）
  并列保留 → 两者均为真，各有适用场景（最常用）
  待用户决定 → 上报用户，不替用户做判断

时间矛盾 → 时间分段，各有适用期
场景矛盾 → 情境标注，分开描述
表里矛盾 → 保留矛盾本身，标记为模式
来源矛盾 → 来源标签，不做裁判

上报触发：
  v vs v 直接矛盾 → 上报
  安全议题 → 上报
  用户要求选择 → 上报
  推断 vs 自认知重大分歧 → 上报
```
