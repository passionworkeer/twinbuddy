# 六问授权协议 — MING Consent Protocol

> 在蒸馏开始之前，必须完成此协议。

---

## 协议说明

创建 MING 灵魂是一个严肃的行为，涉及到真实的人的自我数据。
本协议确保所有相关方都清楚这个 MING 的用途和限制。

## 授权等级判定

### 判定逻辑

```
if (问题1 == "我自己" AND 问题2 == "明确同意") {
  等级 = "A"
} else if (问题2 == "部分知情" OR 问题4 in ["家庭成员"]) {
  等级 = "B"
} else {
  等级 = "C"
}
```

### 等级说明

| 等级 | 名称 | 能力 |
|------|------|------|
| **A** | 完整创建 | 所有功能正常可用 |
| **B** | 受限创建 | 限制部分敏感功能，需要显著标识 |
| **C** | 需复核 | 需要人工复核授权合法性 |

### 本次判定结果

**授权等级**：{A/B/C}

---

## 限制条款（按等级）

### A级 — 完整创建
无额外限制。所有功能可用。

### B级 — 受限创建
- 输出必须包含 `source=MING-restricted` 标识
- 禁止用于公开分享场景
- 禁止商业使用

### C级 — 需复核
- 创建流程暂停
- 提示用户联系支持团队确认授权
- 如果无法确认，拒绝创建

## 授权记录

蒸馏开始前，将此协议的完整回答写入：

```
profiles/{slug}/consent.json
```

### consent.json 格式

```json
{
  "version": "1.0",
  "created_at": "{ISO8601}",
  "subject_type": "{answer to Q1}",
  "consent_level": "{explicit/partial/uncertain/unauthorized}",
  "distillation_scope": "{full/core/custom}",
  "usage_scope": "{private/family/public}",
  "commercial_use": "{prohibited/non-exclusive/exclusive}",
  "expiry": "{unlimited/date/withdrawable}",
  "authorization_level": "A/B/C",
  "limitations": ["{applicable limitations}"],
  "subject_acknowledged": {true/false},
  "subject_relationship": "{relationship if not self}"
}
```

---

## 撤回权

无论选择何种授权等级，用户都有权：
1. 随时撤回同意（发送 `/ming-revoke`）
2. 要求删除已创建的 MING
3. 要求删除所有相关数据

撤回后：
- 删除 `profiles/{slug}/` 目录
- 删除 `souls/{slug}/` 目录
- 向用户确认删除完成
