# PRD：AI即兴剧场（多人小游戏）

## 1. 产品概述与核心价值

**一句话定位：** 一款AI担任叙事导演（GM）的多人即兴角色扮演剧场游戏，4-8名玩家在AI引导下围绕随机关键词即兴共创荒诞故事，AI最终评选"最佳赛博演员"。

**Slogan：** 当AI成了你的即兴搭档，没有人能抢你的戏。

---

## 2. 用户场景与痛点

| 痛点 | 描述 |
|------|------|
| 冷场焦虑 | 即兴表演最大敌人是"不知道该说什么"，普通AI没有结构引导 |
| 角色失焦 | 多玩家同时表演时剧情容易散掉，没有GM维护叙事主线 |
| 评分黑箱 | 传统桌游投票主观且尴尬，AI评分让结果更有仪式感 |
| 关键词植入生硬 | "马"这种随机词如何不破坏体验地嵌入剧情 |
| 多人同步困难 | 如何让每个人都能输入且不被他人看到 |

---

## 3. 产品功能详细设计

### 3.1 角色分配系统

**角色池（8个赛博朋克风格角色）：**

| ID | 角色名 | 性格标签 | 口头禅示例 |
|----|--------|----------|------------|
| C01 | 赛博清道夫-阿柒 | 毒舌+强迫症 | "我不关心，我只收拾" |
| C02 | 数据炼金师-零 | 冷静+隐喻狂 | "一切都是01的排列组合" |
| C03 | 情绪贩子-妙蛙 | 亢奋+表演型 | "来来来！流量要爆炸了！" |
| C04 | 记忆黑客-苔 | 神秘+欲言又止 | "我看见过……算了" |
| C05 | 废弃协议-Dave | 呆萌+直线思维 | "Dave想回家" |
| C06 | 深渊导游-夜游 | 暗黑+诗意 | "欢迎来到你不想去的地方" |
| C07 | 漏洞诗人-泡面 | 荒诞+谐音梗 | "我是漏洞，但不是筛子" |
| C08 | 仿生前任-Aaa | 执念+自嘲 | "我不是bug，我是feature" |

**分配流程：**
1. 输入玩家数量（4-8人）
2. AI随机抽取N个不重复角色
3. 大屏展示角色卡（仅玩家本人看到）
4. 可选"重新抽取"一次（仅一次）
5. 确认后角色锁定，游戏开始

### 3.2 回合机制（10轮叙事）

```
Round N  [场景推进]  AI描述当前场景与NPC行为
    ↓
[角色回合]  每个玩家依次输入台词（限时30秒）
    ↓
[冲突注入]  AI插入转折或关键词事件
    ↓
[回合结算]  本轮评分展示（赛博计分板动画）
    ↓
[剧情摘要]  AI生成半句，下轮玩家接着续写
```

**第5轮关键词节点：**
```
【赛博广播 - 紧急插播】
"检测到未注册实体突破叙事防火墙。关键词：马。
所有叙事单元请在下一回合内对此实体作出响应。"
```

**第10轮高潮+颁奖：**
- 每个角色有"终幕独白"机会
- 总榜金银铜展示
- "最佳赛博演员"获专属AI生成徽章

### 3.3 评分系统

| 维度 | 权重 | 评判标准 |
|------|------|---------|
| 角色一致性 | 50% | 台词是否契合角色性格、口头禅、语言风格 |
| 创意与戏剧性 | 50% | 是否有出人意料的剧情推进、精彩比喻、冲突设计 |

**评分Prompt示例：**
```
【角色扮演评判任务】
角色：{personality}
口头禅：{catchphrase}
本轮场景：{current_scene}
玩家历史台词：{player_history}
玩家本轮台词：{current_line}

评分（1-100分）：
1. 角色一致性
2. 创意与戏剧性

输出：{"score_character": 85, "score_creativity": 90, "total": 87.5, "comment": "一句话评语"}
```

## 4. 技术架构

### 4.1 系统架构

```
┌──────────────────────────────────────┐
│           玩家端（前端）              │
│  大屏展示 + 手机扫码输入（接力模式P0）│
└──────────────┬─────────────────────┘
               │ HTTP轮询（接力模式P0） / WebSocket（P1升级）
┌──────────────▼─────────────────────┐
│        GM Orchestrator              │
│   单一LLM + 三组Prompt模板分发      │
│   - 场景推进Prompt模板              │
│   - 评分Prompt模板                  │
│   - 终局颁奖Prompt模板              │
└──────────────┬─────────────────────┘
               │
    ┌──────────┴──────────┐
    ▼                     ▼
┌─────────┐      ┌─────────────┐
│计时控制器│      │状态存储     │
│(倒计时) │      │sessionStorage│
└─────────┘      └─────────────┘
```

> ⚠️ **技术评审修正：** "三Agent"改为"三Prompt模板"——不要真的部署3个独立Agent服务，用1个大模型+3组Prompt模板+状态分发，延迟更低更可控。

### 4.2 GM状态机（压缩至5轮）

```
IDLE → ROLE_ASSIGN → WAITING_INPUT → SCORING → NARRATIVE →
(Rinse Repeat × 5) → FINALE → AWARD → IDLE
```

> ⚠️ **10轮改为5轮：** 10轮GM Prompt工程需要15-20小时，5轮只需6-8小时，且评委5分钟内看不到10轮的效果，5轮足够展示核心体验。

### 4.3 多人同步方案

**P0默认：接力模式（无网络依赖）**
- 所有玩家在同一台平板上操作
- 每人输入完成后点"提交"，AI汇总后统一展示
- 零同步复杂度，工时：2小时
- **这是黑客松的真实MVP方案**

**P1升级：局域网WebSocket**
- 平板运行Node.js服务器，玩家手机扫码
- 工时：6-8小时
- 仅在接力模式稳定后才启动

## 5. 随机关键词适配

### "马"的四步植入法

**Step 1（第4轮尾）：** "叙事层边缘检测到异常波动……"

**Step 2（第5轮头）：**
```
【赛博广播 - 紧急插播】
"检测到关键词：马。所有叙事单元请在下一回合内对此实体作出响应。"
```

**Step 3（第5-6轮）：** 玩家即兴应对

**Step 4（第6-10轮）：** "马"作为NPC/隐喻/吉祥物持续出现

### "马"的多重语义适配

| 方向 | 示例 |
|------|------|
| 字面 | 真的出现一匹赛博改造马 |
| 谐音 | 骂、吗、妈 |
| 隐喻 | "内卷的野马"、"心里的那匹马" |
| 网络梗 | "小马宝莉"、"马了" |
| 赛博化 | "数据马 DataHorse.exe"、"量子马" |

## 6. MVP范围（40小时·修正版）

> ⚠️ **技术评审修正：** 原版工时严重脱离实际。多人实时系统Bug密度远高于单用户系统，以下为修正后版本。

| 模块 | 原估算 | 修正后 | 优先级 |
|------|--------|--------|--------|
| GM核心Prompt工程（5轮版） | 8h | **10h** | P0 |
| 前端（大屏+接力模式） | 10h | **8h** | P0 |
| 评分Prompt + 展示 | 4h | **4h** | P0 |
| 关键词植入逻辑 | 3h | **3h** | P0 |
| 颁奖典礼（电子奖状，非打印）| 3h | **2h** | P1 |
| WebSocket升级（仅接力稳定后）| 0h | **6h** | P2 |
| 压力测试+修Bug | 6h（全程）| **7h（全程）**| 全程 |

**P0必须上线：**
- [ ] 角色分配 + 角色卡展示（接力模式）
- [ ] 5轮场景生成 → 输入收集 → 评分 → 推进（完整闭环）
- [ ] 双维度评分 + 展示
- [ ] 第3轮强制植入关键词（关键词从第5轮改到第3轮，减少轮次）
- [ ] 大屏展示全流程

> 💡 **三Agent降级说明：** 原"GM Orchestrator + Per-Player Agent + Scoring Agent"三Agent架构改为"1个LLM + 3组Prompt模板"，理由：真实多Agent需要独立服务部署+调用链管理，40小时不可能稳定交付。

## 7. 现场演示脚本（5分钟·修正版）

> ⚠️ **Demo评审修正：**
> 1. "8轮压缩到2分钟"不可行——8轮完整演示最少需要6分钟，改为5轮
> 2. 关键词环节独立展示，不埋在流程里
> 3. 颁奖改为"电子奖状截图"而非当场打印

**团队分工（4人）：**
| 角色 | 职责 |
|------|------|
| 主持A | 引导观众上台+暖场 |
| 操作B | 控制大屏+播放预制视频备用 |
| 观察C | 监控实时系统状态 |
| 备用D | 观众不配合时团队内演示 |

```
0:00-0:30  开场Hook（30秒）
主持A：
"接下来是全场最有互动感的环节——AI即兴剧场。
 我需要请3位观众上台做演员——对，就是你们三位，请上来。"
[如果观众不配合（最常见）：
 "那我们团队先演示，你们坐在台下也可以选角色！"
 关键：永远不冷场，直接开始]

0:30-1:00  角色分配（30秒）
[大屏展示8张角色卡翻转动画]
"1号玩家——赛博清道夫阿柒！"
[3名参与者各看到自己角色（平板遮挡）]
"请念出你们角色的口头禅！"
[破冰，这是评委最放松的时刻]

1:00-2:30  即兴表演（90秒，5轮版）
Round 1（15秒）：AI生成场景，玩家输入台词
Round 2（15秒）：继续推进
Round 3（20秒）【关键词爆发】：AI广播"马！"
Round 4（15秒）：玩家即兴应对"马"
Round 5（25秒）：终幕，每人一句告别语

主持A："Round 1——深夜便利店的诡异交易，开始！"

关键词爆发（最关键30秒）：
"【赛博广播】检测到关键词：马。所有叙事单元，请响应！"
[大屏"马"字赛博动画，全场集中]

2:30-3:30  终幕与颁奖（60秒）
"终幕——每位演员说一句告别语。"
[AI生成剧情结局摘要（20秒朗读）]
"总榜揭晓。本届最佳赛博演员是——"
[电子奖状截图展示，全场颁奖]
# 注意：电子奖状截图，非打印——避免物理设备故障

3:30-4:15  技术架构讲解（45秒）
"底层是GM Orchestrator——
 不是三个独立Agent，是1个大模型+3组Prompt模板，
 控制5轮叙事的节奏、评分和颁奖。
 关键词'马'在第3轮实时注入。"
[架构图同步展示]

4:15-5:00  收尾（45秒）
"谢谢三位演员！你们可以回到座位了。"
主持A：
"AI即兴剧场——让每个人都能当最佳演员。"
[预制视频放在触手可及位置，任何问题立刻切换]
```

> 🔴 **必须提前录制完整预制Demo视频（团队内部录制），放在触手可及的位置。** 一旦现场出任何问题（网络断/观众不配合/台词接不上），立刻切视频，零停顿。

## 9. 风险与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| 多人同步技术事故 | 高 | 提前调试；备用单设备模式；预录Demo视频 |
| 玩家冷场 | 高 | 角色卡附带台词模板；AI提供实时提示按钮 |
| 关键词"马"植入尬演 | 中 | AI提供"马的多重语义"提示面板；幽默兜底旁白 |
| AI评分被质疑 | 中 | 评分标准事前公开；AI评语有理有据 |
| 40小时开发不完 | 持续 | 严格MVP优先级；每天2次彩排 |
