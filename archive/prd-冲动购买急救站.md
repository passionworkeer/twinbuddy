# PRD：冲动购买急救站

## 1. 产品概述与核心价值

**一句话定位：** 抖音信息流中的"消费冷静AI"，在冲动与理性之间，为用户提供5秒钟的情绪诊断报告，让每一次下单都经得起睡前的自己审视。

**反常识主张：** 我们不阻止购买，我们告诉用户"你为什么想买"。通过降低用户冲动消费的事后后悔率，建立对平台的长期信任，最终提升平台的真实成交额和用户LTV。

---

## 2. 用户场景与痛点

**场景A：** 深夜23:40刷到筋膜枪视频，正要下单——卡片弹出

**场景B：** 直播间限量倒计时，主播喊"最后500单"——卡片在购物车按钮旁闪过

**场景C：** 朋友圈种草新款包，晚上想搜同款——刚进搜索结果页，卡片就出现了

| 痛点 | 情绪强度 |
|------|---------|
| 情绪劫持："当时脑子一热，买完第二天就后悔了" | ★★★★★ |
| 社会证明催眠："评论区都在说好，我也想试试" | ★★★★☆ |
| 稀缺焦虑："不买就没了，不抢亏大了" | ★★★★☆ |
| 事后合理化："我都想买了，肯定是需要啊" | ★★★☆☆ |

## 3. 产品功能详细设计

### 3.1 五维冲动分析模型

**维度一：情绪激活指数（EA Index）**
- 信号：视频播放完成率、点赞/评论率、弹幕密度
- 档位：低/中/高 → 冲动系数 0.6 / 0.8 / 1.0

**维度二：价格冲击系数（PI Coefficient）**
- 信号：商品价格 vs 用户历史客单价中位数
- 例：平时买100元衣服的用户刷到800元连衣裙，PI=1.5

**维度三：需求真实性评估（NA Score）**
- 信号：用户近30天相关品类搜索词频、浏览记录、收藏夹
- 例：近期搜过"筋膜枪 放松肌肉"的用户，NA加分

**维度四：社交压力指数（SP Index）**
- 信号：评论区情绪倾向、博主粉丝数与信任度
- 例：头部博主+一边倒叫好 = 高SP

**维度五：冷静期缺失分（CI Deficit）**
- 信号：用户从看到商品到点击购买的时长
- 例：5秒内完成浏览→点击→下单 = CI Deficit 极高

### 3.2 冲动诊断计算

```
冲动指数 = (EA × 0.30 + PI × 0.25 + SP × 0.20 + CI × 0.25) × NA调权
后悔概率 = sigmoid(冲动指数 - 1.2) × 100%
```

## 4. 技术架构

> ⚠️ **技术评审修正：** 原版架构图中出现 Flink 是严重错误。Flink 是分布式流处理框架，部署配置至少需要8-12小时，绝不可能在40小时黑客松内完成并稳定运行。已删除，改用轻量异步方案。

### 4.1 修正后整体架构

```
[用户点击"测一测"]
    ↓
[五维规则引擎 / Python FastAPI] ← MVP主力（规则计算，< 50ms）
    - 五个维度按规则计算得分
    - 规则引擎用简单的 if-else + 加权求和实现
    ↓
[AI解读生成层 / 豆包 API] ← P0提升优先级（文字解读生成）
    - 五维得分向量 → 个性化解读文本
    - 冷静期建议生成
    ↓
[ECharts 雷达图渲染] ← 原WebGL改为ECharts，开发效率提升5倍
    - SVG雷达图 + CSS动画
    - 动态逐维亮起效果
    ↓
[用户端]
    - 悬浮提示 → 全屏卡片 → 报告页
```

### 4.2 五维解读 Prompt（核心资产·新增）

```python
impulse_report_prompt = """
【冲动购买诊断报告生成】

用户当前购买决策数据：
- 情绪激活指数（EA）：{ea_level}（低/中/高）
- 价格冲击系数（PI）：{pi_level}（低/中/高）
- 需求真实性（NA）：{na_level}（低/中/高）
- 社交压力指数（SP）：{sp_level}（低/中/高）
- 冷静期缺失（CI）：{ci_level}（低/中/极高）
- 综合冲动指数：{impulse_score}（0-1）
- 后悔概率预测：{regret_probability}%

生成要求：
1. 为每个维度输出一句15-25字的个性化解读
2. 解读语气：冷静分析 + 一针见血，不鸡汤，不废话
3. 找出得分最高的那个维度，作为"主诉问题"重点解读
4. 最后输出一句"认知偏误点名"（用专业心理学/经济学术语点名用户的认知偏差）
5. 附上一句"冷静期建议"（1小时/24小时/72小时三选一）

输出JSON格式：
{{
    "main_problem": "主诉问题维度名",
    "main_analysis": "主诉问题15-25字解读",
    "dimension_analyses": {{
        "EA": "情绪维度15-25字解读",
        "PI": "价格维度15-25字解读",
        "NA": "需求维度15-25字解读",
        "SP": "社交维度15-25字解读",
        "CI": "冷静期15-25字解读"
    }},
    "cognitive_bias": "认知偏误名称 + 5字说明",
    "cool_down_suggestion": "冷静期建议（10字以内）"
}}

语气示例（对比）：
❌ 差："这个商品看起来很划算呢，你可能是一时冲动哦"
✅ 好："稀缺性触发 + 社会认同压力 = 你现在的'划算感'是别人营造的，不是你真正需要的"
```

### 4.2 MVP技术方案

**MVP阶段用规则引擎替代ML模型：**

| 维度 | 规则 |
|------|------|
| EA | 视频播放完成率 > 80% = 高 |
| PI | 价格 > 历史均值2倍 = 高 |
| NA | 近30天有搜索记录 = +0.2修正 |
| SP | 评论"已入手"占比 > 60% = 高 |
| CI | 加购时长 < 10秒 = 极高 |

**为什么MVP用规则引擎反而更好：** 可解释性强，能向用户和评委清晰讲清楚"你为什么想买"的逻辑

## 8. 为什么能拿奖

**评委关注点 vs 我们的方案：**

| 评委关注点 | 我们的优势 |
|-----------|-----------|
| 高情绪价值 | 击中"冲动消费后悔"普遍痛点，共情门槛极低 |
| Agent技术架构 | 规则引擎（可解释性）+ 豆包AI生成层（文字解读）|
| 现场Demo丝滑度 | 10秒出报告，5分钟讲完整条链路 |
| 创新差异化 | 反常识叙事——"帮用户省钱反而帮平台赚钱" |

**反常识Pitch是拿奖关键：**
- 让产品从"工具"升维为"平台级价值主张"
- 评委的问题都已被预判和回答
- 记忆点是清晰的逻辑链，不是功能堆砌

**技术选型说明（评委可能问）：**
- 为什么五维不用机器学习模型？→ 可解释性 > 复杂度，黑盒模型无法向用户解释"你为什么想买"
- 为什么不用Flink？→ Flink是分布式系统，40小时黑客松用asyncio足够；我们选择把时间花在Prompt工程而非运维
