# PRD：孪生搭子 TwinBuddy

> 版本：v1.2
> 日期：2026年4月17日
> 适用：抖音赛季黑客松（40小时，4人团队）
> 核心理念：**你的完整人格数字孪生体，替你找遍全世界的好搭子**

**一句话定位：** 不是算法匹配，是人格全权代理——两个数字孪生体以"另一个你"的身份当面协商，过程实时可见，结论自带说服力。

**Slogan：** 找一个真正懂你的搭子，不需要问卷，不需要算法——只需要另一个你。

---

## 1. 产品背景与愿景

### 1.1 背景

当前搭子匹配依赖标签筛选（ENFP、爱旅行、可接受早起），本质上和"关键词搜索"没有区别。用户填了问卷，算法返回结果，双方对彼此的认知仍然停留在表面。

问题不在匹配算法，在于**数据本身就不够完整**。

用户人格数据散落在各处：抖音行为、MBTI测试、AI聊天记录、个人简介、微信/小红书对话……单一平台数据无法完整还原一个人。传统的搭子匹配永远只能拿到用户愿意填写的那部分，永远是残缺的画像。

### 1.2 核心洞察

> **真正懂你的搭子，不靠问卷，靠另一个你。**

当你的数字孪生体和对方的数字孪生体当面协商时，它们：
- 用你的语气说话
- 按你的决策风格权衡
- 知道你的边界（什么情况会拒绝）
- 记得你提过的每一个偏好

对方感知到的不是"一个AI在匹配"，而是"另一个你在和我谈"。

### 1.3 长期愿景

每个人都拥有一个完整人格的数字孪生体，它：

- **持续进化**：通过多源数据（抖音 + 用户上传 + 聊天记录 + 更多平台）持续训练，精准模拟真实性格、决策风格、说话习惯、兴趣深度
- **全场景覆盖**：旅行搭子、兴趣同好、职场导师、情感陪伴、语言交换、游戏队友……所有交友场景一网打尽
- **自主协商**：Agent间可跨平台协商、组局、甚至帮用户婉拒不合适的邀约
- **用户主权**：用户可随时"喂养"数字人（上传新聊天记录 → 实时进化），也可随时删除

### 1.4 与MING Skill的关系

孪生搭子的**人格蒸馏引擎**直接基于已有的 MING Skill（E:\desktop\hecker\MING\）构建：

- 9步灵魂锻造流程（identity → values → reasoning → boundaries → voice → relationships → growth）
- 证据驱动的持续进化
- 7维度可验证的人格 distillation
- 用户自述后亲自判断"像不像"的验证机制

**MING Skill 是产品差异化壁垒，不是附加功能。** 没有MING，数字人只是标签合集；有了MING，数字人才是"另一个你"。

## 2. 目标用户与用户画像

### 2.1 核心用户

| 维度 | 描述 |
|------|------|
| 年龄 | 18-35岁Z世代 |
| 行为特征 | 爱社交、爱分享、多平台活跃 |
| 技术接受度 | 高——MBTI爱好者、AI重度用户、喜欢跟AI深度对话 |
| 平台足迹 | 抖音、小红书、微信、AI聊天产品（ChatGPT、Kimi等） |

### 2.2 用户痛点

| 痛点 | 描述 |
|------|------|
| 问卷疲劳 | 已经填了无数搭子问卷，每次都一样无聊 |
| 算法失真 | 算法觉得"ENFP+爱旅行=合适"，但实际性格合不合完全不知道 |

- 随时补充数据（上传新聊天记录、新的AI对话）

- 最终决策权仍在用户，但前期沟通成本为零

## 3. 核心功能详细设计

### 3.1 多源数据输入

**设计理念：** 数据来源不限格式、不限平台，用户有多少就给多少，数字人"像你"的程度随数据量提升。

**MVP支持的数据源：**

| 数据源 | 格式 | 说明 |
|--------|------|------|
| MBTI | 文字输入（四维字母组合 + 可选详细描述） | 最快速的人格框架输入 |
| 个人简介/Bio | 文字输入 | 用户自述，主观真实 |
| AI聊天记录 | TXT / JSON（从任意AI产品复制粘贴） | 行为真实，说话方式、决策风格直接提取 |
| 抖音数据 | JSON（点赞、评论等）Mock数据 | 展示数据源扩展性，MVP用Mock |
| 照片 | 图片上传 | 用于Avatar生成 |

**输入界面设计：**
- 引导页：展示支持的5种数据源，每个带格式说明
- 上传区：拖拽或点击上传，支持多文件同时上传
- 实时反馈：上传后显示"已识别X条MBTI维度 / Y段对话 / Z个偏好"
- 隐私提示（醒目展示）："上传数据仅用于本次数字人生成，不会永久存储"

**数据预处理（后端）：**
```
输入文件 → 格式检测 → 提取器（MBTI解析器 / 聊天记录解析器 / Bio提取器）→ 统一JSON
```

### 3.3 数字人自述验证

**核心交互：** 数字人生成后，先以第一人称自述人格，用户判断"像不像"。

**自述文本示例：**
```
我是这样的一个人——
我讨厌计划被打乱，但比起计划，我更讨厌无聊。
和朋友出去，我通常是那个临时改目的地的人。
我不会主动拒绝，但如果对方触碰了我的底线，我会沉默而不是争吵。
和我旅行，最好别问我"你想吃什么"，因为我的答案永远都是"你定"。
```

**验证界面：**
- 数字人自述文字展示（带打字机效果）
- 下方两个按钮：**"像"** / **"不太像"**
- 点击"像" → 进入协商流程
- 点击"不太像" → 触发MING进化流程，引导用户补充/修正数据

**MVP冷启动：** 内置3-5个Mock人格（ENFP旅行博主 / ISTJ金融男 / INFP自由插画师等），评委可选择直接体验双数字人对聊，无需上传自己的数据。

**界面布局（核心UI）：**

```

│   你的数字孪生体       │   对方数字孪生体       │
│   [Avatar 头像]       │   [Avatar 头像]       │
│                      │                      │
│  [聊天气泡 ←]         │  [→ 聊天气泡]         │
│  [聊天气泡 ←]         │                      │
│                      │  [→ 聊天气泡]         │
│  [聊天气泡 ←]         │  [→ 聊天气泡]         │
│                      │                      │
│  兼容度：78%          │  潜在冲突：2处         │
└──────────────────────┴──────────────────────┘
              [协商进度条]
         [兼容度报告生成区域]
```

**Avatar设计（2D卡通头像）：**
- 每个数字人配一个AI生成的卡通头像（基于用户照片或人格标签生成）
- 头像旁边显示人格标签（e.g., "ENFP · 直觉型 · 冷幽默"）
- 对聊过程中，气泡颜色区分双方（你的：蓝色 / 对方：橙色）

**协商状态机（LangGraph）：**
```
IDLE → PERSONA_INIT → CHAT_ROUND → CONFLICT_DETECTED →
NEGOTIATION → CONSENSUS_FOUND → REPORT_GENERATED
```

- 每个CHAT_ROUND：数字人双方各输出一轮对话
- CONFLICT_DETECTED：检测到人格冲突（旅行节奏 / 预算 / 作息），触发协商策略
- NEGOTIATION：数字人基于人格画像中的values和boundaries进行协商
- CONSENSUS_FOUND：达成共识或确定不可调和的分歧

**对聊内容方向（旅行搭子场景）：**
```
数字人A：我更倾向于说走就走，不想提前订太死的行程
数字人B：我需要提前知道每天住哪，临时找酒店我会焦虑
→ 协商：A同意提前订住宿，B同意不锁死每天具体行程
→ 共识：行程大框架提前定，细节弹性
```

### 4.5 隐私设计

| 原则 | 实现方式 |
|------|---------|
| 数据不过留 | 所有上传数据仅用于本次会话，会话结束即清理 |

| 临时存储 | Session级别存储，不落持久化数据库 |
