## 一、产品概述

### 一句话介绍

**TwinBuddy** — 你的数字孪生体，在后台替你找搭子。

### 核心定位

不是找搭子的工具，而是「刷到就懂你的瞬间」。

**Slogan：** 你的另一个你，已经替你搞定了。

---

## 二、核心功能

| 功能 | 说明 |
|------|------|
| **数字孪生体生成** | 从 MBTI + 兴趣标签 + 一句话描述 + 城市，蒸馏出完整人格画像 |
| **抖音风格 Feed** | 模拟抖音上下滑体验，发现潜在搭子 |
| **懂你卡片** | 协商完成后呈现最优搭子方案，无需再看其他 |
| **双 Agent 协商** | 两个 AI 替你谈判行程细节，节省沟通成本 |
| **六维雷达图** | 行程节奏/美食偏好/拍照风格/预算控制/冒险精神/随性程度 |
| **地点导览卡** | 提供精选目的地导览，辅助决策 |

## 三、技术架构

### 核心模块

```
┌─────────────────────────────────────────────────────────────┐
│                     TwinBuddy 技术栈                          │
├─────────────────────────────────────────────────────────────┤
│  前端：React 18 + TypeScript + Tailwind CSS + Recharts     │
│  后端：FastAPI + LangGraph + MiniMax LLM                    │
│  部署：Railway (后端) + Vercel (前端) + HTTPS               │
└─────────────────────────────────────────────────────────────┘
```

### 关键文件

| 模块 | 路径 | 说明 |
|------|------|------|
| 后端入口 | `main.py` | FastAPI 应用，端口 8000 |
| 人格蒸馏 | `persona_generator.py` | LLM prompt 约束系统 |
| 融合引擎 | `fusion_engine.py` | 多维度人格向量融合 |
| 卡片引擎 | `card_engine.py` | 懂你卡片触发逻辑 |
| 协商状态机 | `negotiation/graph.py` | LangGraph 实现 |
| 前端 Feed | `twinbuddy/frontend/src/pages/FeedPage.tsx` | 抖音风格页面 |
| 前端引导 | `twinbuddy/frontend/src/pages/OnboardingPage.tsx` | MBTI + 兴趣引导 |

## 五、页面流程

### Onboarding 引导页

```
Step 1: MBTI 选择 (16 型人格)
    ↓
Step 2: 兴趣标签 (可多选)
    ↓
Step 3: 一句话描述 + 语音输入
    ↓
Step 4: 目的地城市选择
    ↓
生成数字孪生体人格文档
```

### Feed 体验页

```
上下滑动浏览视频/搭子卡片
    ↓
触发懂你卡片 (部分视频会触发)
    ↓
查看协商结果 (可展开三层详情)
    ↓
确认/拒绝搭子方案
```
