# TwinBuddy v2 设计文档

> 日期：2026年4月17日
> 状态：已批准

## 背景

v1 PRD 与赛道三"AI体验：刷到懂你的瞬间"存在错位——v1设计是"主动操作型工具"，赛道三是"被动信息流内容单元"。v2改造核心：**从"主动找搭子"变为"被动懂你卡片"**。

## 核心设计决策

### 产品形态
独立 Web App（打包成二维码扫码即用），内嵌模拟抖音 Feed 界面。评委扫码 → 刷 feed → 懂你卡片弹出 → 点击深入 → 雷达图 → 同意进真人聊天。

### 数据路径（两条并存）
- **被动触发**：Mock 数据 + 预设人格池，评委无需上传任何数据即可体验完整流程
- **主动输入**：用户上传 MBTI/聊天记录 → MING 蒸馏 → 生成用户自己的数字孪生体

### 卡片内容（三层结构）
1. **第一层**（刷到即见）：地点 + "你的孪生体已帮你谈好了" + 两个数字人对话截取（2-3句）
2. **第二层**（点击展开）：完整协商对话记录 + 兼容度雷达图 + red_flags
3. **第三层**（点击同意）：成功提示 → 进真人聊天流程

### 对方数字人策略
- **Demo期**：4个预设Mock人格（ENFP旅行博主/ISTJ金融男/INFP插画师/ENTJ产品经理）
- **未来**：真实用户接入，架构不变

### 技术架构（不变）
- 前端：React + Vite + Tailwind（新增 Feed + 卡片组件）
- 后端：FastAPI + MING蒸馏 + LangGraph协商
- LLM：通义千问/Kimi（JSON Mode）
- 存储：Session级，不持久化

### Demo场景聚焦
旅行搭子场景为主，Feed 里放 3-5 条 Mock 视频，第 3-5 条为懂你卡片触发点。

## 已确认决策

1. Demo方式：独立 Web App + 模拟抖音 Feed（不用真实抖音生态）
2. 卡片内容深度：文字 + 数字人对话截取 + 雷达图（不做 Avatar 动画，降级为静态头像）
3. 对方人格：预设 Mock 人格为主，支持未来真实用户扩展
4. 场景：旅行搭子为主，Demo 可选展示美食/球赛扩展性
