# PRD：孪生搭子 TwinBuddy v3

> 版本：v3.0
> 日期：2026年4月17日
> 适用：抖音赛季黑客松（40小时，4人团队）
> 赛道：赛道三 · AI体验：刷到懂你的瞬间
> 核心理念：**刷到懂你的瞬间——你的数字孪生体已在后台替你谈好了，点个头就能出发。**

**一句话定位：** 不是主动找搭子的工具，而是刷抖音时自然出现的一张懂你卡片——两个数字孪生体已经替你协商完毕，结果直接端到你面前。

**Slogan：** 你的另一个你，已经替你搞定了。

---

## 1. 产品背景与愿景

### 1.1 赛道核心要求

赛道三原文：

> "创造一种新的信息流内容单元……在被刷到的那一刻，把你想要的信息、情绪与选择，都刚好准备好。"

### 1.2 产品理念

**从"主动找搭子"变为"被动懂你卡片"。**

用户正常刷抖音，突然一张 AI 卡片弹出来——你的数字孪生体已经帮你和对方聊过了，行程排好了，搭子也筛出来了，你只需要点"同意"。

关键理念转变：
- ~~给你一个工具，你自己造数字人找搭子~~
- **你刷视频时，AI 已经偷偷帮你找好搭子、协商完、排出行程了，你只需要点头**

### 1.3 长期愿景

每个人都拥有一个完整人格的数字孪生体，它：
- **持续进化**：通过多源数据持续训练，精准模拟真实性格、决策风格、说话习惯
- **全场景覆盖**：旅行搭子、美食搭子、赛事搭子……所有"需要找个伴"的场景一网打尽
- **被动触发**：数字孪生体在后台持续运行，当检测到用户需求信号时，自动弹出协商结果卡片
- **主动汇报**：用户可主动进入"我的搭子"页面，查看数字孪生体帮你聊了哪些搭子

### 1.4 技术核心：MING 人格蒸馏引擎

基于 MING Skill 构建的数字孪生体：
- 9步灵魂锻造流程（identity → values → reasoning → boundaries → voice → relationships → growth）
- 四维蒸馏（cognition / expression / behavior / emotion）
- 证据驱动的持续进化
- 固定权重融合（MBTI 0.9 > 说话风格示例 0.8 > 兴趣标签 0.7）

## 5. TwinBuddy Skill 框架（核心架构）

### 5.1 Fusion Engine（融合蒸馏）

固定权重融合，数据不足时标记 `confidence: "low"`：

| 数据源 | 权重 | 说明 |
|--------|------|------|
| MBTI | 0.9 | 性格骨架，最高权重 |
| 说话风格示例 | 0.8 | 直接决定语气风格 |
| 兴趣标签 | 0.7 | 结构化偏好 |

### 5.2 四维蒸馏

| 维度 | 说明 |
|------|------|
| cognition | 认知模式、思考方式 |
| expression | 表达风格、措辞习惯 |
| behavior | 行为模式、决策风格 |
| emotion | 情感反应、情绪边界 |

### 5.3 Isolation（数据隔离）

- 前端 localStorage 持久化
- 刷新页面仍能看到之前生成的数据
- 关闭浏览器清空
- 无需后端数据库/登录

### 5.4 LangGraph Agent（双数字人协商）

使用 LangGraph 状态机控制协商流程：

```
IDLE → PERSONA_INIT → CHAT_ROUND → CONFLICT_DETECTED →
NEGOTIATION → CONSENSUS_FOUND → REPORT_GENERATED → CARD_TRIGGERED
```

**协商流程：**
- 3个搭子Agent（提案/评估/让步/拒绝）
- 最多3轮协商
- 预生成结果兜底

### 5.5 Persona Engine

生成完整 persona JSON：
```json
{
  "identity": { "self_description": "...", "roles": [...], "core_unchangeable": "..." },
  "values": [{ "value": "...", "weight": 0.95, "example": "..." }],
  "reasoning": { "style": "...", "process": "...", "blind_spots": [...] },
  "boundaries": { "hard_limits": [...], "preferences": [...] },
  "voice": { "vocabulary": [...], "sentence_pattern": "...", "humor_style": "...", "samples": [...] },
  "relationships": { "attachment_style": "...", "trust_patterns": "..." },
  "growth": { "growth_edges": [...] }
}
```

同时生成 Rive 2D 头像描述词（3D 不做）。

### 8.3 核心 API 接口

**POST /onboarding** — 保存用户开局数据
```json
请求：
{
  "mbti": "ENFP",
  "interest_tags": ["说走就走", "爱拍照", "重度火锅党"],
  "speech_sample": "我其实不太在意去哪，你能帮我选吗",
  "target_city": "成都"
}

响应：
{
  "success": true,
  "confidence": "medium",
  "persona_summary": "..."
}
```

**GET /feed/{video_id}** — 获取Feed内容
```json
响应：
{
  "video": { /* Mock视频数据 */ },
  "card_triggered": true,
  "card_content": {
    "location": "川西 · 3天2夜草原线",
    "persona_a_summary": "弹性随性型",
    "persona_b_summary": "计划周密型",
    "negotiation_result": { /* 协商结果 */ },
    "compatibility_report": { /* 兼容度报告 */ }
  }
}
```

**POST /negotiate** — 双数字人协商一轮
```json
请求：
{
  "persona_a": { /* 用户人格画像 */ },
  "persona_b": { /* 对方人格画像 */ },
  "scenario": "川西3天2夜草原线"
}

响应：
{
  "round": 1,
  "message_a": "我其实对行程没那么执念...",
  "message_b": "太好了，那大框架就按你说的来？",
  "conflict_detected": false,
  "compatibility_score": 76
}
```

**POST /agree** — 用户同意搭子
```json
请求：
{ "card_id": "xxx" }

响应：
{ "success": true, "message": "已发送搭子邀请" }
```

## 11. 风险与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| 数据不足幻觉 | 高 | 固定权重 + confidence标记 + 大量预生成Mock |
| 语音输入兼容性 | 低 | 可选按钮，默认文本输入 |
| 视频播放卡顿 | 中 | 预下载本地视频 |

| 数据持久需求 | 中 | localStorage（刷新可见，关闭清空） |
| 评委质疑"不是真正抖音" | 中 | 明确说 Demo 展示形态，真实落地在抖音生态内 |
