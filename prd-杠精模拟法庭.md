# PRD：杠精模拟法庭

## 1. 产品概述与核心价值

### 产品是什么

"杠精模拟法庭"是一款嵌入信息流场景的AI辩论互动卡片。当用户在刷到带有争议性、话题性的视频时，屏幕下方会以"法庭传票"的形式弹出辩论卡片，邀请用户与AI扮演的"杠精法官"进行3轮正面对线。AI从不脏话，却能用逻辑和刻薄让人气笑——它的目标是制造"明明很气但忍不住想继续回击"的体验。

### 核心价值

| 维度 | 价值点 |
|------|--------|
| 用户 | 释放表达欲，满足对抗欲，在安全环境里体验辩论快感 |
| 平台 | 延长用户停留时长，提升评论区活跃度，制造社交传播素材 |
| 技术 | 展示Agent人设一致性、长程记忆、多轮对话的工程能力 |
| 评委 | 高情绪浓度 + 可视化对话 + 现场Demo冲击力 |

### 产品定位一句话

**让每一次刷到争议视频的愤怒，都变成一场让人上瘾的辩论游戏。**

---

## 2. 用户场景与痛点

### 核心场景

**场景A：刷到明显站队的观点视频**
用户刚刷到一个"女人就应该做饭"或"996是福报"类视频，评论区已经炸了。用户心里有话想说但懒得打字，或者写了也没人看。

**场景B：和朋友一起刷，看到离谱观点**
两个人一起刷视频，对某个观点集体无语，但只能口头吐槽，没有地方释放。

**场景C：单纯想玩，想看AI能杠到什么程度**
用户带着猎奇心态，想测试AI的"杠技"上限。

### 用户痛点

| 痛点 | 描述 |
|------|------|
| 愤怒无处安放 | 看到离谱观点想反驳，但没有合适出口 |
| 评论没人看 | 认真写了评论沉入信息流，0反馈很扫兴 |
| 无聊刷视频 | 纯刷视频15分钟后开始空虚，手指机械滑动 |
| 辩论对手太弱 | 评论区对线体验差，要么对方开始骂人，要么逻辑漏洞百出 |

## 3. 产品功能详细设计

### 3.1 卡片交互流程

```
信息流 → 触发检测 → [法庭传票弹出] → 用户点击"开庭"
     → 争议摘要展示 → 用户选立场（正/反） → 3轮辩论
     → 终局宣判 → 生成"战报"分享卡片 → 退出或再来一局
```

**触发机制：**
- 视频话题标签含敏感词（预设词表：性别对立、职场压榨、明星八卦等）
- 视频评论区情绪指数超过阈值
- 用户主动点击评论区"挑战"按钮

**传票卡片设计：**
- 视觉风格：法庭传票 + 红色印章 + "挑战者XXX已出庭"
- 动画：卡片从底部滑入，带轻微震动效果
- 停留时间：3秒后显示"开庭/划走"两个按钮

### 3.2 辩论机制设计

**轮次结构（3轮）：**

| 轮次 | 时长 | AI行为 | 用户输入 |
|------|------|--------|----------|
| 第1轮 | AI先手 | 抛出争议点 + 极端立场 | 选择立场（正/反）后输入观点 |
| 第2轮 | 各45秒 | 反驳 + 举"例子"（虚构场景） | 继续对线，输入论点 |
| 第3轮 | 各60秒 | 终极抬杠 + 挖逻辑漏洞 | 最终陈述 |

**每轮规则：**
- 用户输入字数上限：200字
- AI每轮输出字数：50-100字（刻薄精炼）
- 超时处理：倒计时归零后自动进入下一轮
- 中途退出：允许，但记录"逃兵"标签

### 3.3 判决系统

**判决维度：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 逻辑严密性 | 30% | 有无自相矛盾 |
| 攻击有效性 | 30% | 是否打到痛点 |
| 幽默值 | 20% | 有没有让人气笑的瞬间 |
| 坚持立场 | 20% | 3轮是否前后一致 |

**判决结果分级：**
- 🥇 **大律师**：总分 >= 85，用户完胜
- ⚖️ **平手**：总分 60-84，互有攻防
- 🗣️ **杠精获胜**：总分 < 60，AI完胜

## 4. 技术架构

### 4.1 整体架构

```
用户端（前端卡片）
    ↕ WebSocket / SSE
辩论Agent服务
    ↕ HTTP
LLM调用层（GPT-4o / Claude Sonnet）
    ↕
辩论状态管理（Redis）
    ↕
关键词挂载引擎（适配层）
```

### 4.2 Agent人设设计

核心人设prompt片段：

```
你扮演"杠精法官"，一个辩论技巧极其高超但绝不使用脏话的人。
你的风格特征：
- 语气：自信、略带嘲讽、逻辑严密
- 常用句式：
  * "你说的这个问题，恰恰证明了..."
  * "我们来捋一捋你刚才的逻辑链..."
  * "你说的对，但只对了一半——因为你忽略了..."
  * "有意思，那我们假设你的前提成立，然后..."
- 绝不使用：脏话、人身攻击、种族/性别歧视词汇
- 始终保持：逻辑自洽、举例具体、态度轻蔑但不越界
- 幽默感：会在反驳中穿插冷笑话或讽刺
```

### 4.4 通用关键词挂载引擎

关键词注入不是硬塞，而是作为辩论语境增强：

```python
# 通用关键词注入框架（适配任意关键词）
def build_keyword_injection(keyword: str) -> str:
    return f"""
[额外指令] 本场辩论如有出现与"{keyword}"相关的任何元素，
请在反驳/举例时自然关联。关联方式任选：
- 字面提及（直接引用该词）
- 谐音/双关（如"马"可关联"骂/吗/妈"）
- 隐喻借用（"像{kw}一样…"）
- 象征延伸（该词代表的精神/行为）

要求：关联必须服务于论点逻辑，不能生硬插入。
关键词不出现时正常辩论，不要刻意制造关联。
"""
```

## 5. MVP范围与优先级

### MVP必须包含（P0）

| 功能 | 说明 | 验收标准 |
|------|------|---------|
| 卡片触发 | 争议视频检测 + 卡片弹出 | 至少能手动触发 |
| 辩论引擎 | 3轮对话 + 状态管理 | 上下文不丢失 |
| AI人设 | 刻薄但干净的语气 | 5个不同话题都能保持人设 |
| 判决系统 | 3轮结束后的评分 + 评语 | 评分有区分度 |
| 战报生成 | 静态卡片生成 | 可截图分享 |

### 暂不做（P2）

- 多语言辩论
- 辩论录音回放
- 排行榜系统
- 实时对战（双人）

## 6. 现场演示脚本（5分钟·修正版）

> ⚠️ 原版脚本存在根本性时间错误：3轮辩论最少需要4.5分钟，1.5分钟绝不可能。以下为可执行版本。

**团队分工（4人）：**
| 角色 | 职责 |
|------|------|
| 主讲A | 操作手机+念预设台词+控制节奏 |
| 主讲B | 负责技术架构讲解（接在辩论后） |
| 控场C | 引导评委参与关键词环节+预备应对AI翻车 |
| 技术D | 监控网络/API状态，预备切换兜底 |

**演示流程（精确到秒）：**

```
【0:00-0:20】开场Hook（20秒）
主讲A：
"你们有没有这种感觉——刷到一个视频，气得想反驳，
  但写了评论没人看，算了。
  今天我要展示的产品，就是让你把这口气给出出来。"
[切换到大屏：法庭主题封面]

【0:20-0:45】触发演示（25秒）
"我们来现场演示。打开这个视频——"
[点开预设争议视频，30秒内播放完]
"视频播完了，没有评论区辩论机会。
  但我们的卡片来了。"
[法庭传票卡片弹出，'开庭'按钮高亮]
"大家看到了，从视频播完到卡片弹出，不到1秒。"

【0:45-2:30】辩论演示（105秒）★ 最关键环节
# 时间分配：Round1=30s, Round2=35s, Round3=35s, 切换=5s
[点击"开庭"，争议摘要展示5秒]

Round 1（30秒）：
- AI先手（5秒）：展示hook_question
- 用户选择立场（正/反），主讲A念预设台词（15秒）
  # 预设台词提前准备，声音要有情绪起伏
- AI反驳（流式输出，边出边读，10秒内读完）

Round 2（35秒）：
- 主讲A念Round2预设台词（15秒）
- AI反驳（15秒）
- 评委可以选择"加一个例子"让AI继续

Round 3（35秒）：
- 主讲A念最终陈述（15秒）
- AI终极反驳（15秒）
- 倒计时5秒提示

【2:30-3:00】终局宣判（30秒）
"现在，终局宣判。"
[判决页面展示：大律师/平手/杠精获胜]
[展示战报卡片，包含：
  - 高光摘要（最精彩2句话）
  - 判决 + 幽默评语
  - "再战一局"按钮]
"大家可以看到，判决是即时生成的，有评分，有评语，有截图分享。"

【3:00-3:45】关键词演示（45秒）
控场C："现在，请评委随意给一个关键词——"
评委说"马"。
[重新开局，注入关键词"马"]
"我们看，带有关键词'马'之后，AI全程把马融入了辩论。"
[快速播放1轮完整辩论（压缩到45秒），重点展示马相关的比喻]

【3:45-4:45】技术架构讲解（60秒）
主讲B（冷静输出）：
"底层架构：状态机管理3轮上下文 + 判决Prompt评分系统。
关键技术点：
1. 流式输出：AI第一个字出来就开始显示，
   评委感知延迟从5秒降到0.3秒。
2. 判决Prompt：4维度量化评分 + AI生成评语，
   保证评分既有区分度又有情绪价值。
3. 关键词挂载引擎：任意随机词都能融入辩论逻辑。"
[架构图在大屏同步展示]

【4:45-5:00】收尾（15秒）
"杠精模拟法庭——让愤怒变成快乐。
谢谢大家。"
[战报卡片二维码展示，提示评委可扫码体验]
```

**兜底方案（必须准备）：**
| 故障场景 | 应对 |
|---------|------|
| AI回复超时 > 5秒 | 自动切换预制回复（提前存3个fallback话术）|
| AI越界（脏话） | Moderation API实时过滤，触发后AI说"这轮我选择不回应" |
| 网络断 | 赛前预录完整辩论视频，切换播放并说明"这是离线模式" |
| 评委不配合 | 改为团队内部演示，评委观看，但强调"可以随时打断提问" |

**预设台词包（必须提前准备）：**
| 话题 | Round1 | Round2 | Round3 |
|------|--------|--------|--------|
| 买房 | "买房是年轻人最好的投资" | 反驳AI"租房更灵活"的例子 | 最终陈述：风险与自由的关系 |
| 彩礼 | "彩礼是传统，应该保留" | 反驳"彩礼是物化"的逻辑 | 最终陈述：情感与金钱的关系 |
| 加班 | "996是福报，年轻就该奋斗" | 反驳"身体最重要" | 最终陈述：奋斗与生活的边界 |

> ⚠️ 每条预设台词不超过80字，现场念的时候要有停顿和情绪起伏，不要像背书。

## 8. 风险与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| AI越界（脏话/歧视） | 高 | Prompt禁止列表 + Moderation API过滤 |
| 冷启动延迟 > 5秒 | 中 | 流式输出 + 预加载动画 + 缓存fallback |
| 争议点抽取质量差 | 中 | MVP用预设话题表，不用自动抽取 |
| 随机关键词适配翻车 | 低 | 赛前准备10个关键词的prompt模板 |
| 评委审美疲劳 | 低 | 法庭主题UI差异化 + 现场选词互动 |
