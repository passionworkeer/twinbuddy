## 一句话

你在刷抖音，另一边的「你」已经在后台帮你把搭子聊好了。

---

## 这不是一个找搭子的工具

普通搭子工具：你填问卷、等消息、自己跟人聊。

**TwinBuddy 不是这样。**

你什么都不用做。你的数字孪生体——另一个你——在后台持续运行。它知道你是什么样的人：你是早起赶行程还是睡到自然醒，你是看见美食走不动道还是随便吃点就行，你是喜欢热闹还是需要独处空间。

然后它帮你去找，去谈，去吵出一套行程来。

等到你刷到那张懂你卡片的时候，两个数字人已经替你把所有事情搞定了。你只需要点一个「同意」。

---

## 为什么这件事有意义

### 每个现代人都在为「找人」这件事付出巨大的隐性成本

- 发消息等回复
- 反复确认行程
- 行程不合吵起来
- 一个人去了，发现完全不合拍

这个成本在旅游搭子场景里最集中，但你认真想想——学习搭子、健身搭子、演唱会搭子，甚至日常找人吃顿饭，哪一个不是这样？

我们不是在发明一个新功能。我们是在解决一个真实存在的摩擦。

### AI Agent 的真正价值，不是帮你回答问题

现在大多数 AI 是「你问，它答」。

但未来的 AI 应该是「你不用问，它替你做」。

你的数字孪生体，不是一个聊天机器人。它是你的一个分身——帮你分析，帮你决策，帮你替你说话。

## 数字孪生体是什么

我们用 **MING 人格蒸馏框架** 把一个人的数据，转化为一个「懂你」的数字人。

### MING 是什么

MING 把一个人拆成四个维度：

| 维度 | 问的问题是 |
|------|-----------|
| **Cognition（认知）** | 你怎么想事情？你是逻辑派还是感觉派？ |
| **Expression（表达）** | 你怎么说话？你是直接型还是委婉型？ |
| **Behavior（行为）** | 你怎么做事情？你是计划狂还是随性流？ |
| **Emotion（情感）** | 你怎么反应？你是容易激动还是波澜不惊？ |

数据越丰富，这四个维度就越准确。

**在 Demo 阶段**，我们用 MBTI + 说话风格 + 兴趣标签来推断这四个维度——这是粗粒度的，但已经够用。

**未来接入抖音数据之后**，数字人会精准得多：你点赞了哪些视频、评论了什么、收藏了什么、搜索了什么——这些行为数据才是真正的人格矿。

### 两个数字人怎么「谈」

我们用 **LangGraph 状态机** 控制两个数字人的协商流程：

```
提案 → 评估 → 让步/拒绝 → 共识/继续协商 → 输出结论
```

最多三轮。每一轮都是真实的来回，不是简单匹配。

协商完了，卡片弹出来，你会看到：你们的节奏合不合拍、预算差多少、哪里可能有冲突、建议怎么磨合——这些全是从两个人的数字人里算出来的。

## 未来愿景：每个人都有一个数字分身

我们真正在做的，不是旅游搭子这个功能。

**我们在押注一个方向：**

> 五年到十年后，每个人都会有一个自己的数字孪生体。
>
> 它了解你的一切——你的性格、习惯、说话方式、决策风格、社交偏好。
> 它在后台持续运行，帮你处理一切「需要找人」的事。
> 它比你更了解你自己。

### 这个数字分身能干什么

**帮你找人**
不只是搭子——学习搭子、健身搭子、饭搭子、演唱会搭子、甚至找合租、找队友、找人一起做项目。

**帮你谈**
它用你的方式替你说话。你是直接型，它不会绕弯子。你是委婉型，它不会太冲。它知道你的底线在哪里，哪些可以让，哪些不能让。

**帮你决策**
当两个人有分歧的时候，它能告诉你：你们的核心矛盾在哪里、折中方案是什么、长期来看合不合拍。它不是给你一个分数就走，它是给你分析。

**帮你维持关系**
数字分身不只在你「找人」的时候出现。它知道你跟谁关系好、谁最近没联系了、什么时候该主动问一句。这个维度现在还没有人做，但它是真实的用户需求。

### 为什么这件事发生在抖音

因为抖音有最多的真实人格数据。

你在抖音上留下的每一次互动——点赞、评论、收藏、搜索、观看时长——都是在告诉平台「我是谁」。这些数据比任何问卷都更真实、更丰富。

把这些数据用来训练你的数字孪生体，这件事只有抖音能做。

而如果数字孪生体出现在抖音的信息流里——你不需要下载新 App，不需要改变任何使用习惯，搭子功能就是抖音的一部分。

## 评分体系：MING 六维度

两个人合不合拍，我们用六个维度来算：

| 维度 | 最高分 | 看什么 |
|------|--------|--------|
| 行程节奏 | 25 | 暴走派 vs. 慢悠派 |
| 社交能量 | 20 | 社交型 vs. 独处型 |
| 决策风格 | 20 | 理性派 vs. 感性派 |
| 兴趣对齐 | 25 | 共同喜好 / 共同厌恶 / 直接冲突 |
| 预算兼容 | 15 | 消费观念是否一致 |
| 人格互补 | -5~+10 | MBTI 四象限 + 互补对 |

总分 0-100，超过 80 算高度契合，低于 50 要谨慎。

每一张卡片里，你不只是看到分数——你看到的是这六个维度上的对比图，冲突的地方用红色标注，优势的地方用绿色标注。

## 项目结构

```
twinbuddy/
├── agents/                      # 核心算法
│   ├── scoring.py              # MING 六维度评分引擎
│   ├── matching_graph.py       # LangGraph 协商图
│   ├── buddy_agent.py         # 搭子数字人
│   └── buddies/               # 100 个搭子人格 JSON
│       └── _archived/          # 废弃文件
│
├── frontend/                    # React + Vite + Tailwind
│   ├── src/
│   │   ├── pages/             # Onboarding / Feed / Result / MatchReport
│   │   ├── components/        # TwinCard / RadarChart / immersive-feed
│   │   ├── hooks/             # useLocalStorage / useOnboarding
│   │   ├── api/               # API 客户端
│   │   ├── types/             # TypeScript 类型定义
│   │   └── __tests__/         # 单元测试
│   └── e2e/                   # Playwright E2E 测试
│
├── backend/                    # FastAPI
│   ├── main.py                # 入口
│   └── api/frontend_api.py    # 前端对接 API
│
└── docs/
    ├── QNA.md                 # 评委 Q&A 手册
    └── _archived/             # 废弃文档
```

## 快速开始

```bash
# 后端
cd twinbuddy/backend
pip install -r requirements.txt
python -m uvicorn main:app --reload --port 8000
# → http://localhost:8000/docs

# 前端
cd twinbuddy/frontend
pnpm install && pnpm dev
# → http://localhost:5173
```
