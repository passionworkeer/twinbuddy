1. 产品背景与愿景（v1.1更新）
1.1 背景
用户人格数据散落在各处：抖音行为、个人简介、MBTI测试、AI聊天记录、微信/小红书对话……单一平台数据无法完整还原一个人。
我们通过多元数据融合，让每个用户拥有一个高度逼真的完整人格数字孪生体，它不仅能“找搭子”，还能自主处理一切交友场景。
1.2 长期愿景（未来方向，明确强化）
每个人都拥有一个完整人格的数字孪生体，它：

通过多源数据持续训练（抖音 + 用户上传 + 聊天记录 + 未来更多平台），精准模拟真实性格、决策风格、说话习惯、兴趣深度。
能24小时自主处理任何交友方案：旅行搭子、兴趣同好、职场导师、情感陪伴、语言交换、游戏队友……
Agent间可跨平台协商、组局、甚至帮用户婉拒。
核心技术路径：统一数据摄入层（Data Ingestion Pipeline） + RAG/Embedding融合 + 动态Persona Refinement（参考MBTI + Big Five + 聊天历史），实现“越用越像我”的持续进化。

黑客松MVP：专注旅行搭子场景 + 多元数据输入演示，快速验证“多源数据 → 完整数字人 → Agent协商”闭环。赛后无缝扩展全场景。

2. 目标用户 & 用户画像（不变，但数据输入更灵活）
核心用户：18-35岁Z世代，爱社交、爱分享多平台数据。

新增支持：MBTI爱好者、AI重度用户（喜欢跟AI深度聊天绘画/规划的人）。
痛点补充：不想只给抖音数据；希望手动补充MBTI、聊天记录，让数字人“更像自己”。

4. 产品架构 & 技术方案（v1.1更新）
架构图（文字版）：
多元数据源（抖音API + 用户上传MBTI/聊天记录/照片）
→ 数据融合层（LLM + Embedding/RAG）
→ 完整人格画像JSON + Skill
→ 火山引擎/Three.js 3D数字人绑定
→ LangGraph多Agent状态机
→ 前端实时可视化
技术栈（40小时最稳）：

前端：React + Vite + Tailwind + @react-three/fiber
3D数字人：火山引擎虚拟数字人API（支持3D卡通、实时唇同步、表情驱动，字节系最匹配）
后端&Agent（你一人负责）：
FastAPI
LangGraph（状态机）+ CrewAI
LLM：通义千问 / Kimi（JSON Mode）
新增：简单文件解析（MBTI文本、聊天记录 → Persona JSON）

数据：MVP全本地Mock/上传；赛后接抖音开放平台（官方API：get-user-like-number、评论等，需授权）
隐私：所有上传数据仅本次会话使用，用户可随时删除“我的数字人”。

关键Prompt模板更新（你后端直接用）：

多元数据融合Prompt：输入[抖音点赞 + MBTI: ENFP + 简介 + 聊天记录片段] → 输出完整{ personality: "...", travel_style: "...", mbti_influence: "...", skills: [...] }
Agent角色仍保持，但输入包含完整Persona。


5. 非功能需求（强化隐私与灵活性）

隐私：明确告知“上传的MBTI/聊天记录仅用于本次数字人生成，不会永久存储”；支持“一键删除我的数字人”。
性能：多源生成 < 10秒。
演示友好：准备4个预设场景（纯抖音、纯MBTI+聊天、混合、失败案例）。
